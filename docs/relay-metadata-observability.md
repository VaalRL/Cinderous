# Relay 可觀測元資料：攻擊面與降低槓桿（紀錄文件）

> **性質**：這是一份**紀錄文件**，非決策。彙整「一個惡意 relay 部署者實際能觀測到哪些
> 元資料」以及「有哪些降低槓桿、各自的代價與衝突」，供未來決策參考。已採納的項目
> 另立 ADR；其餘為**暫緩／記錄**。
>
> **決策對照**：
> - **(d) 在線 jitter/隱身、(e) 心跳 P2P 卸載** → 已採納，見 `docs/adr/0088`。
> - 其餘（M1 盲化收件匣、(g) Tor 傳輸、(h) 外層填充盤點、(a/b/c/f) 結構性方案）→ 本文件記錄，暫緩。
> - 前向保密（內容層 harvest-now-decrypt-later）屬另一議題，見 `docs/adr/0028`（維持）與 `0091`（MLS 暫緩）。

## 前提

Gift Wrap（NIP-17/59，ADR-0002）已藏掉**訊息內容**（NIP-44）與**寄件者身分**。本文件只談
**Gift Wrap 之後仍殘留**、relay 看得到的元資料。威脅模型（PRD §6）已將中繼營運者列為頭號
對手、並把「全球被動流量分析對手」列為 out-of-scope——本文件的降低槓桿以「抬高惡意 relay
的關聯成本」為目標，不承諾對抗國家級全域對手。

## 一、攻擊面：relay 今天實際看得到什麼

| # | relay 可觀測 | 現況為何藏不掉 | 嚴重度 |
| --- | --- | --- | --- |
| **M1** | **收件人 pubkey** | kind 1059 的 `p` tag＝收件人真實 pubkey；客戶端以 `#p` 拉自己的收件匣（`relay/src/relay-core.ts` inbox 邏輯）。**relay 知道「誰在收訊、何時收」** | 高（Gift Wrap 補不到的最大洞） |
| **M2** | **上線時序 + 真實 pubkey** | 心跳 kind 20000 以真實金鑰明簽廣播（非 Gift Wrap） | 中（已由 ADR-0088 部分處理） |
| **M3** | **連線 IP** | 對 relay 的 WebSocket 連線來源 | 中 |
| **M4** | **訊息大小 / 時間 / 頻率** | 事件經 relay 轉發／儲存 | 低～中 |
| **M5** | **同 IP 多 pubkey 的關聯** | 連線層可把多裝置／多身分連結 | 中（跨身分不可連結是 ADR-0055 的目標，但網路層仍可能） |
| **M6** | **AUTH 簽章證據**（開 `requireAuth` 時） | 客戶端送真實金鑰簽章的 AUTH 事件（`relay-core.ts` handleAuth） | 低～中（取捨見 ADR-0057） |

**核心洞察**：**M1 + M2 的時序關聯**——「X 在線的同時某收件匣在收訊」——讓惡意 relay（或被串
策展進清單的一組 relay）能**部分重建社交圖譜**，即使 Gift Wrap 已藏寄件者。M1 是最有價值的目標，
也最難，因為它撞本專案兩根柱子（見第三節）。

## 二、降低槓桿一覽（含代價與衝突）

### M1 收件人 pubkey（最有價值、最難）

- **(a) 盲化輪替收件標籤**：不以真實 pubkey 當 `p` tag，改用寄收雙方共享秘密**按時間衍生的
  輪替標籤**；relay 只見一堆無法互相關聯、也連不到單一 pubkey 的隨機標籤。
  - 代價：收件人須同時訂閱「每聯絡人 × 每 epoch」多個衍生標籤（**訂閱膨脹**，撞 ADR-0006 容量模型）。
  - 衝突：**破壞「只憑 npub 就能加你／傳你」**——陌生人算不出你的輪替標籤（撞 ADR-0024 QR 加好友／
    加好友請求流程）。
  - 可能折衷：**已互加聯絡人走盲化、陌生首次接觸走公開 npub** 的混合式（待評估）。
- **(b) k-匿名分桶**：訊息依收件人 key 的粗前綴分桶，收件人拉整桶本地過濾，relay 只知「你是桶中 K 之一」。
  - 代價：**頻寬放大約 K 倍**，直接壓免費層。
- **(c) 誘餌流量**：對隨機收件人發假 Gift Wrap 模糊收訊圖。
  - 代價：同樣吃免費層額度。

### M2 上線時序

- **(d) jitter + 隱身上線** → **已採納（ADR-0088）**。
- **(e) 心跳／在線狀態 P2P 卸載**（複用 ADR-0006 F5）→ **已採納（ADR-0088）**。
- **(f) 盲化在線信標**：在線狀態亦以輪替金鑰廣播（同 (a) 家族）。藏得更徹底，但同樣有訂閱膨脹／
  加好友衝突 → **暫緩**。

### M3 連線 IP

- **(g) 選用 Tor／mixnet 傳輸**：對 relay 的 WebSocket 走 Tor，relay 只見出口節點 IP。
  - 代價：延遲、行動端電量 → 設為「高隱私模式」選項、預設關即可 → **暫緩、記錄**。
  - 註：既有 TURN 遮蔽的是「P2P 對端看到的 IP」，**不等於** relay 看到的；(g) 補的是不同洞。

### M4 流量分析

- **(h) 固定尺寸填充盤點**：NIP-44 本身已對長度做填充（分桶到近似 2 次方）；待**盤點 Gift Wrap
  外層是否亦對齊**。低成本 → **記錄、待盤點**。

### M6 AUTH

- **(i) AUTH 設為選用 / 一次性證明**，或以 (a) 盲化收件匣根本免在 filter 亮真實 pubkey。取捨與
  ADR-0057 相關 → **記錄**。

## 三、關鍵取捨（為何 M1 不輕啟）

真正藏住收件人（M1），與本專案兩個賣點**直接對立**：

1. **「只憑 npub 就能加好友／傳訊」** 需要**穩定、可被陌生人計算**的收件地址；盲化輪替標籤讓
   陌生人算不出。
2. **「永久免費」** 需要 relay 端能靠 `#p` **精準投遞**、不做全桶拉取；k-匿名／誘餌／全下載都以
   頻寬（＝免費層額度）換匿名集。

因此「藏收件人 = 犧牲其一」。這與前向保密（ADR-0091）暫緩同一種性質：**不是做不到，是要拿核心
價值去換**。任何 M1 方案在推進前，都應先回答「如何與 npub 加好友＋免費層共存」（混合式 (a) 是
目前看來最有機會的方向）。
