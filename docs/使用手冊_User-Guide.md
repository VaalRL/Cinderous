# Cinder 使用手冊

> 給第一次使用去中心化即時通訊的你。這份手冊聚焦在「Cinder 和 LINE / Messenger 這類軟體**不一樣**的地方」——這些不一樣不是刁難，而是「沒有一家公司掌管你的帳號與訊息」所必須付出的代價，換來的是**沒有人能偷看、封鎖或刪除你的對話**。

---

## 0. 先理解一件事：這裡沒有「帳號」

LINE 有一台中央伺服器，記著「這支手機號碼＝這個帳號」，你的訊息、好友、頭貼都存在他們那裡。所以你換手機、忘記密碼，都能靠「手機簡訊驗證」把帳號要回來——因為 LINE 隨時能證明「你是你」。

**Cinder 沒有這台伺服器。** 你的身分不是一組帳號密碼，而是一把**只有你持有的鑰匙**（技術上叫「私鑰」，長得像 `nsec1...`）。這把鑰匙：

- **由你的裝置產生，從不上傳給任何人**——所以沒有任何公司、政府、駭客能從伺服器偷走它。
- **就是你的身分本身**——誰拿到它，誰就是你。反過來說，弄丟它、又沒有備份，就沒有任何「找回帳號」的客服能救你。

這一條想通了，後面所有「為什麼要備份」「為什麼換手機這麼講究」就都順了。

| | LINE | Cinder |
| --- | --- | --- |
| 你的身分是 | 綁在手機號碼的帳號 | 一把你獨有的鑰匙（私鑰 nsec） |
| 誰保管你的資料 | LINE 公司的伺服器 | **只有你的裝置**（訊息不上雲、除非你自己開啟） |
| 換手機靠 | 手機簡訊驗證 | 備份碼／雲端備份／裝置配對（本手冊 §5） |
| 忘記密碼靠 | 重設密碼 email | 你的私鑰備份（本手冊 §4） |

## 1. 第一次使用：登入畫面

打開 Cinder，你會看到兩個欄位：

- **顯示名稱**：別人看到的名字。**這只是個標籤**，不是帳號——可以跟別人重複，也可以隨時改。它**不能**拿來登入或找回身分（找你的是下面那把鑰匙，不是名字）。
- **中繼站網址（Relay）**：通常**已自動填好一座可用的中繼站**，你直接登入即可，不用打字。想換別座就改掉它；清空則進示範模式（本機玩玩、不連真實網路）。

> **「中繼站」是什麼？** 它是幫你和好友「轉交訊息」的信箱——當對方不在線上，你的訊息（**已經加密**）會暫存在這裡，等對方上線再取走（最多存 7 天）。中繼站**只看得到密文**，看不到內容，也看不到你跟誰在聊。你可以用官方的、朋友自架的、或自己架一台（見 `docs/self-hosting-raspberry-pi.md`）。

按「登入」後，Cinder 會**在你的裝置上產生一把全新的鑰匙**——這就是你的新身分。第一次就這麼簡單，複雜的部分都是「之後怎麼保護與搬移這把鑰匙」。

> ⚠️ **重要**：這一步產生的是**全新身分**。如果你是要在新手機上「登入舊身分」，**不要**用這個畫面重新登入（那會變成一個跟舊的完全無關的新人）——請看 §5「換裝置」。

## 2. 加好友：用鑰匙，不是用電話簿

LINE 靠手機號碼或掃 LINE ID 加好友。Cinder 沒有電話簿，所以加好友靠**對方的公開鑰匙**（叫 `npub1...`，是私鑰對應的「公開版」，給出去很安全）。

三種方式：

1. **貼上對方的分享字串**：對方在自己的 App 複製「我的 ID」，傳給你，你貼上即可。字串可能長這樣 `npub1...` 或 `npub1...@wss://對方的中繼站`（後面那段是告訴你的 App「去哪座中繼站找他」）。
2. **掃 QR code**：對方出示 QR，你掃（行動端）。
3. 加好友後，雙方第一次通訊時會**自動交換顯示名稱**，所以你不必手動輸入對方的名字。

> 為什麼要交換「中繼站位置」？因為每個人可能用不同的中繼站。你的 App 會自動記住「這位好友在那座站」，之後訊息就直接送到對的地方；就算記錯了，系統也有備援會繞路送達。這些都是自動的，你通常不用管。

## 3. 日常使用：跟你熟悉的一樣

傳訊息、貼圖、表情、已讀、群組、語音視訊通話、傳檔案——這些操作和一般通訊軟體沒有差別，直接用就好。差別只在「幕後」：

- 每一則訊息、每一次通話，都是**端到端加密**的。中繼站與任何第三方都只看到亂碼。
- 訊息**優先存在你自己的裝置**。這是隱私的好處，但也代表「換裝置時資料不會自動出現在新手機」——這正是 §5 要解決的。

## 4. 本地密碼：共用電腦時的額外保護（可選）

**如果只有你一個人用這台裝置，可以跳過這節。** 本地密碼是給「多人共用同一台電腦」的情境。

### 它保護什麼

沒設本地密碼時，任何能打開你這台電腦、進到你 Windows 帳號的人，都能直接進你的 Cinder 看訊息。設了之後，你的私鑰與本機資料會被密碼**加密**——沒有密碼，就算把整台電腦搬走也解不開。

### 怎麼設

**設定 →「安全（本地密碼）」→「啟用本地密碼」**。過程中會**要求你先備份私鑰**（見下方），並輸入兩次新密碼。啟用後，每次打開 App 會先出現解鎖畫面。

還有兩個進階選項：
- **隱藏此身分**：啟用密碼後可勾選，讓這個身分**連出現在切換清單裡都不會**——共用電腦上，別人連「你有這個身分」都不知道。之後用身份列的 🔒 鈕輸入密碼喚回。
- **閒置自動上鎖**：離開電腦 5 分鐘沒操作，自動鎖回去。

### ⚠️ 忘記本地密碼怎麼辦？

這是最需要理解的一點。密碼**不是拿去比對的**，它是「解開你資料的鑰匙原料」——所以系統**根本沒有存你的密碼**，也**無法「重設」或「寄一封重設信」**。這聽起來很可怕，但有救援路：

**忘記密碼時，在解鎖畫面點「忘記密碼？」，輸入你的私鑰（nsec）或加密備份碼，設一組全新密碼**——就能救回這台裝置上的**完整資料**（不是重來，是原封不動地打開）。

換句話說：**你的私鑰備份，就是你的「忘記密碼」保險。** 這也是為什麼啟用密碼前一定要你先備份。

> 但如果你**連私鑰備份都弄丟了、又忘記密碼**——那這台裝置上的本地資料就真的永久打不開了（沒有任何後門）。所以：**私鑰備份請務必收好**。

## 5. 換裝置 / 換手機：三條路，看你要救回多少

這是 Cinder 跟 LINE 差最多、也最需要事先準備的地方。LINE 換裝置一鍵搞定，是因為你的資料都在他們伺服器上。Cinder 的資料在**你自己手上**，所以你得選一種方式把它搬過去。

**先記住一個關鍵詞：私鑰（nsec）＝你的萬能鑰匙。** 它能在任何新裝置還原你的身分。以下三條路，差別在於「除了身分，還能一起搬回多少資料」。

### 路線 A：加密備份碼（推薦當作保險）

**是什麼**：把你的私鑰用一組**你自訂的備份密碼**加密，打包成一段字串＋一張 QR。你自己保管（列印、存隨身碟、存你信任的雲端都行）。

**怎麼產生**：設定 →「身分備份」→「產生加密備份碼」→ 設定備份密碼。

**怎麼還原**：新裝置的「新增身分」貼上備份碼＋輸入備份密碼。

**救回什麼**：身分本身 ＋ 你的中繼站位置。**聯絡人與訊息不會跟著這條路回來**（那要靠 B 或 C）——但你的身分活了，好友傳訊時會慢慢重新出現。

> 這是「最後一道防線」，強烈建議每個人都做一份。它比直接抄私鑰安全（多一層密碼），弄丟也只是那一份失效，身分不受影響。

### 路線 B：加密雲端備份（換裝置最像 LINE 的體驗，但預設關閉）

**是什麼**：把你的**加密**狀態（聯絡人、群組、封鎖清單、設定，甚至近期訊息）存一份在你的中繼站上。新裝置登入後**自動拉下來、幾秒還原**。中繼站全程只看到密文。

**怎麼開**：設定 →「雲端備份（加密）」→ 選一種模式：
- **關閉**（預設）：什麼都不上傳，隱私最大。
- **基本**：聯絡人、群組、封鎖清單、設定（不含訊息內容）。
- **完整**：基本 ＋ 近期訊息。

**怎麼還原**：新裝置用「備份碼＋密碼」登入後，快照會自動合併進來。

**取捨**：這是最接近 LINE「換裝置自動還原」的體驗，代價是你的加密狀態會**存在中繼站上**（雖然是密文）。所以它**預設關閉**、由你決定要不要開。關閉時會**立刻刪除**中繼站上的那份備份。

> 誠實提醒：備份由你的**身分私鑰**保護，不是本地密碼。任何拿到你私鑰的人本來就能讀你的訊息，所以這不算多開一個洞——但如果你連「近期訊息」都不想留在中繼站，就選「基本」或「關閉」。

### 路線 C：裝置配對（兩台都在手邊時，搬得最完整、最私密）

**是什麼**：舊裝置與新裝置**直接連線**（點對點，完全不經過中繼站），把**全部資料**——包括完整歷史訊息——搬過去。

**怎麼做**：
1. 舊裝置：設定 →「配對新裝置」→ 出現一段配對碼與 QR。
2. 新裝置：登入畫面 →「從舊裝置匯入」→ 貼上配對碼。
3. 兩台會各自顯示一組 **4 位數字**。**確認兩邊數字一樣**，在舊裝置按「相符，開始傳送」。
4. 完成後新裝置就是舊裝置的完整複製。

> 那 4 位數字（叫 SAS）是防「有人半路攔截配對碼冒充你的新裝置」。**數字不一樣就一定有問題，請按「不符」中止。** 這一步花你三秒，但擋掉了最危險的攻擊。

**取捨**：搬得最完整、內容完全不經中繼站（最私密），但**需要兩台裝置同時在手邊、都開著**。適合「舊手機還在、換新手機」的情境。

### 三條路怎麼選

| 情境 | 建議 |
| --- | --- |
| 平常就該做的保險 | **A 備份碼**（做一份收好） |
| 舊裝置還在、要搬去新裝置 | **C 配對**（最完整） |
| 舊裝置已遺失/損壞，但之前開過雲端備份 | **B 備份**（自動還原） |
| 舊裝置已遺失、沒開雲端，只有備份碼 | **A**：身分回來了，好友靠他們傳訊逐漸重現，近 7 天訊息從中繼站補回 |

> **完全弄丟、什麼備份都沒有？** 那身分無法還原（沒有任何客服能救，因為根本沒有伺服器存你的鑰匙）。這就是為什麼**現在就去做一份備份碼**。

## 6. 換中繼站

想換一座中繼站（例如原本用的關站了、或你想換到朋友自架的），**不需要換身分**。

**設定 → 中繼站區塊 →「更換中繼站」→** 輸入新網址。App 會：
- 保留你的身分、聯絡人、全部歷史（只是換了「信箱」）。
- 自動通知你的好友「我搬到新站了」，他們之後會把訊息送到新站。
- **不漏信保障**：更換中繼站後 7 天內，App 會**自動**繼續去舊站補收「還沒改道的好友送來的訊息」，確保不漏信。這是全自動的、你不需要做任何事（也不會顯示任何東西）。

> 這跟「重新登入」完全不同。重新登入＝變成新人；更換中繼站＝同一個你、換個信箱。系統設計成前者絕不會意外發生。

## 7. 一台裝置、多個身分

你可以在同一個 App 裡有多個身分（例如「工作」和「私人」），用畫面上方的**身份切換列**切換。每個身分的資料完全隔離，互相看不到，別人也**無法看出這兩個身分是同一個人**（這是刻意的隱私保護）。

- 用身份列的 **＋** 新增身分（可產生全新的，或貼上私鑰／備份碼匯入既有的）。
- 搭配 §4 的「隱藏身分」，某些身分可以完全藏起來。

## 8. 常見問題（FAQ）

**Q：我只是想聊天，為什麼要懂這麼多？**
A：日常聊天你什麼都不用懂，跟 LINE 一樣用就好。這份手冊的複雜部分，全都是為了「換裝置」和「保護鑰匙」——做好一件事就能高枕無憂：**去設定產生一份加密備份碼收好**（§5 路線 A）。

**Q：一位好友突然從清單消失了？**
A：通常是暫時的。好友清單靠「收到對方訊息」重建，對方一傳訊或一上線廣播就會回來。單向好友（你加他、他沒加你）可能要重貼一次他的 npub。

**Q：忘記本地密碼？**
A：解鎖畫面點「忘記密碼？」，用私鑰或備份碼救回並設新密碼（§4）。前提是你有備份。

**Q：換手機但舊手機已經壞了/丟了？**
A：看你之前做了什麼備份——有雲端備份就自動還原（§5-B），只有備份碼就還原身分、資料靠好友重現與 7 天補收（§5-A）。**都沒有就無法還原**，所以請務必事先備份。

**Q：我的訊息真的沒人看得到嗎？**
A：對。端到端加密，中繼站與任何第三方都只看到密文。唯一能讀的是你和對話對象。代價就是「沒有一台伺服器幫你保管」，所以備份要自己來。

**Q：群組換手機後不見了？**
A：群組比較特別。如果你是**群主**，你的資料裡就有群組、跟著你搬。如果你是**成員**，群組會在**群主下次上線**時自動把你加回來（群主的 App 會定期廣播群組名單）；7 天內漏掉的群訊也會在群組恢復後補收。

**Q：企業/公司內部使用？**
A：Cinder 有企業模式——公司自架封閉中繼站、由管理者派發成員名冊、可強制某些政策（例如禁止檔案傳輸、通話走公司伺服器）。這部分由公司管理者設定，一般成員照常使用即可。

## 9. 名詞對照表

| Cinder 用語 | 大白話 / LINE 類比 |
| --- | --- |
| 私鑰（nsec1…） | 你的萬能鑰匙＝身分本體。**最高機密，等於你的一切。** |
| 公鑰 / npub（npub1…） | 你的「ID」，給別人加你用，公開無妨。 |
| 中繼站 / Relay | 幫忙轉交加密訊息的信箱（可換、可自架）。 |
| 顯示名稱 | 只是個標籤，非帳號、可重複、可改。 |
| 本地密碼 | 這台裝置的鎖（共用電腦才需要）。 |
| 加密備份碼 | 你的「換裝置／忘記密碼」保險，自己保管。 |
| 雲端備份 | 存在中繼站的加密狀態，換裝置自動還原（要自己開）。 |
| 裝置配對 | 兩台裝置面對面直接搬全部資料。 |

## 10. 一句話總結

**Cinder 把「你的資料由你掌控」做到底——代價是「換裝置與備份要自己負責」。**
新手只要記得一件事：**現在就去設定裡產生一份加密備份碼，收在安全的地方。** 其餘的，跟你平常用的通訊軟體沒兩樣。
