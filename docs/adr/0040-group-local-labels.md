# 0040. 群組本地標籤與置頂（自訂標籤）

- 狀態：已接受
- 日期：2026-07-04
- 相關文件：docs/adr/0027（群組聊天加密方案）；docs/adr/0038（url-hygiene 本地偏好持久化寫法先例）；apps/desktop UI

## 背景與問題

M9 群組聊天（ADR-0027）目前群組只有一個固定 `name`，UI 無法讓使用者整理群組清單（分類、置頂）。使用者需要「自訂標籤」來分類群組。

此需求有兩種本質不同的做法：

- **本地個人標籤**：只有自己看得到，用來整理清單；純客戶端、不進協定。
- **共享群組標籤**：群組狀態的一部分、全員同步；需新增帶內控制訊息與收斂處理。

## 決策

採 **本地個人標籤（純客戶端）**：

- 標籤與置頂是**個人檢視偏好**，只存本機、**不進 Nostr 事件、不扇出、不觸及中繼**，與 core `Group`（協定權威狀態）完全解耦。
- 持久化沿用既有先例（ADR-0038 的 `url-hygiene.ts`）：直接以 `localStorage`（鍵 `nb.groupPrefs`）存 `groupId → { labels, pinned }`；node 測試以 stub 覆蓋。
- 純邏輯收斂於 `apps/desktop/src/ui/group-labels.ts`（正規化、加/移標籤、置頂、過濾、排序、載入/存檔），與 React 解耦、以 TDD 覆蓋。
- UI：群組列顯示可移除的標籤 chip、附加標籤輸入、置頂鈕；群組區上方提供標籤過濾列。置頂者排前、同組維持原序。
- 群組離開/解散時 `pruneGroup` 清除其偏好，避免殘留。

**不採共享群組標籤**（本次）：其需新增 `group-set-tags` 控制訊息、管理者權威與多設備收斂（ADR-0009），複雜度高；且「分類/置頂」本質是個人偏好，本地即足。共享群主題/可改群名列為未來項（屆時另立 ADR，順帶補 `group-rename`）。

## 理由

- **零協定/隱私風險**：不動加密路徑、不新增中繼狀態，完全符合「本地優先、明文不出裝置」硬規則。
- **最大複用、最小新面**：持久化與 UI 皆沿用既有樣式與 `localStorage` 先例；核心邏輯純函式、易測。
- **與 SSOT 一致**：群組身分/成員仍由後端（ADR-0027 帶內狀態）權威維護；標籤只是疊加的本地檢視層，不製造平行真實來源。

## 後果

- 正面：使用者可即時分類/置頂群組整理清單；不影響任何成員、不需伺服器改動；`group-labels.ts` 15 項單元測試 + `ContactListWindow` 4 項渲染測試涵蓋。
- 負面 / 已知限制：
  - 標籤**不跨設備**（本機限定）。未來若要同步，可走 ADR-0009 自封同步（可選、後續）。
  - 僅群組適用；聯絡人標籤為後續可延伸項。
- 後續行動：如需「全員共享群主題／可改群名」，另立 ADR 定義 `group-rename`/`group-set-tags` 控制訊息與收斂語意。
