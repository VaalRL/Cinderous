# 0240. 真隱身：隱身時也停止訂閱聯絡人心跳

- 狀態：已接受
- 日期：2026-07-23
- 相關文件：ADR-0088（既有隱身：停止廣播）、ADR-0237（元資料可連結性——presence 訂閱洩漏聯絡人集合；
  本 ADR 是其 **Tier 1 的具體實作形式**）、ADR-0129（無內容心跳信標）、ADR-0109（自適應心跳）、
  ADR-0180（離職接管建構即隱身）、`docs/SECURITY.md`

## 背景與問題

「隱身」在 Cinderous **已經存在**（ADR-0088 (d)，`setInvisible`）：開啟時 `beat()` 與 presence 廣播
早退 → **完全不廣播在線狀態**（中繼與聯絡人都看你離線）。但它只做了**一半**：

- **發布側**（你報自己在線）：關掉 ✅
- **訂閱側**（你問中繼「誰在線」）：**照舊開著** ❌

而訂閱側送的正是 `{kinds:[20000], authors:[你的聯絡人]}`——ADR-0237 指出的核心洩漏：**把你的
聯絡人集合、以真名交給中繼**。所以既有「隱身」**名不符實**：你對**聯絡人**隱身了，但對**中繼**
完全沒有——中繼照樣知道你有哪些聯絡人。對一個叫「隱身」的功能，這是個隱私洞。

## 決策

**把隱身補完成「真隱身」：隱身時發布側與訂閱側都關。**

- **訂閱側**（`subscribeOn`）：`invisible` 時**不放入心跳 filter**——`authors:[聯絡人]` 不再送給中繼。
  其餘 filter（收件匣 `#p:自己`、typing/nudge、快照、信令…）不動，**訊息收發完全不受影響**。
- **切換**（`setInvisible`）：`resubscribe()` 重建合併 REQ（隱身移除心跳 filter／復出加回）。
  進入隱身時另 `presence.clear()` ＋ `emitContacts()` → 聯絡人**立即**在你眼中變離線（不等 60s 逾時）。
- **仍收 P2P presence**：`invisible` 只閘**你送出**的 presence，不閘**接收**。故你**正在對話、P2P 已通**
  的聯絡人，其在線狀態仍透過資料通道顯示（與 ADR-0237 討論的嚴格模式 UX 一致）。

這是 **ADR-0237 Tier 1（嚴格 presence 隱私）的具體落地**——但**折進既有的「隱身」**，而非另立
並存的模式（少一個概念、也順手補上「隱身」名不符實的洞）。

## 理由

- **為何折進「隱身」而非新增模式。** 既有隱身已是一個 opt-in 開關、已接 UI、已被離職接管（ADR-0180）
  使用。它缺的正是訂閱側；補上即為真隱身。新增並存名稱只會與「隱身」語意撞、徒增困惑。
- **為何接受失去「顯示離線但仍偷看」（lurk）。** 經典 MSN 的 appear-offline 讓你隱身卻仍看得到別人。
  但那對隱私是**假的**——你還在訂閱，聯絡人集合照樣外流。放棄它換得名副其實的隱身，划算；且「隱身」
  字面就是「把自己藏起來」，本就該對中繼也藏。
- **為何純客戶端、無 relay/協定改動。** 隱身只是**送更少**：不送心跳訂閱、不發心跳。中繼看你就像
  「一個沒有聯絡人的人」，**分不出你在隱身**。也因為是「送更少」，**IP 關聯拆不掉它**（沒有東西可
  關聯）——這是它勝過 crypto 層混淆的地方（ADR-0237 的核心結論）。
- **為何清 presence。** 停訂閱後不再收心跳，但既有「在線」記錄會殘留到逾時窗（~60s）。清乾淨再重繪，
  讓「進入隱身」即時反映（清單變灰）。

## 後果

- 正面：
  - **隱身名副其實**：對第三方中繼，你的聯絡人集合**不再外流**；發布側早已關（別人看你離線）。
  - **ADR-0237 Tier 1 落地**，且是**唯一不被 IP 關聯打穿**的客戶端手段（送更少）。
  - 改動局部：`subscribeOn` 一個條件 ＋ `setInvisible` 重訂閱/清 presence ＋ `PresenceTracker.clear()`。
    訊息收發、離職接管（ADR-0180）皆不受影響。
- 負面 / 已知殘餘風險：
  - **行為改變**：既有隱身使用者原本「顯示離線仍能看別人」，升級後**也看不到聯絡人在線**
    （清單變灰）——UI 文案（`settings_invisibleHint`）已更新說明取捨。
  - **presence 體驗 ≈ 關閉**：隱身時只看得到「正在對話且 P2P 已通」的聯絡人（ADR-0237 討論的取捨）。
  - **仍非完整元資料隱私**：發布側停了，但**收件匣訂閱**（`#p:自己`）與**發訊連線**仍綁真名——
    連線歸屬（ADR-0237）與時序殘餘不因隱身消失。真正完整的元資料隱私仍是**自架**。
- 後續行動 / 待辦：
  1. 更新 `ARCHITECTURE.md §5`（心跳「隱身」語意）與 `docs/SECURITY.md`（隱身作為 opt-in 緩解）。
  2. ADR-0237 標記 Tier 1 已由本 ADR 實作；並據前次量測發現下修「P2P 卸載對訂閱洩漏的緩解」措辭
     （P2P 只覆蓋正在對話者、且不從心跳訂閱移除他們）。
