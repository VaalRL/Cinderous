# 0246. 官網預設語言改英文、預設淺色主題、新增企業版頁、技術原理擴充

- 狀態：已接受
- 日期：2026-07-24
- 相關文件：ADR-0235（官網 SEO/資訊架構）、ADR-0187（官網資訊架構改版）、ADR-0090（官網雙語）、ADR-0241（中繼分片）、ADR-0242（多裝置 CRDT）、ADR-0243（WebRTC TURN 保底）、ADR-0245（前向保密上線硬閘：文案不得宣稱 FS）

## 背景與問題

官網（`apps/website/`）為預渲染、雙語、SEO/GEO 導向的靜態站。近期有四項需求：

1. **預設語言與主題**：官網原本預設走繁中根路徑（`/` = zh-Hant）＋預設深色（夜森林）。要改為**預設英文＋預設淺色**，以主打國際受眾、並用更中性的第一印象。
2. **技術原理過於精簡**：近幾次已落地多項機制（中繼分片、多裝置 CRDT 同步、WebRTC TURN 保底、換機備援），但技術原理頁未反映，顯得不夠詳細。
3. **缺企業版介紹**：FAQ 有一題帶到企業模式，但官網沒有一個可被索引、可分享、講清楚企業部署的入口。
4. **FS 的紅線**：前向保密（ADR-0245）尚未通過外部審計，上線硬閘明訂**官網文案不得宣稱 FS**——擴充技術原理時不得把 FS 寫進去。

## 考量的選項

- **預設語言**
  - A：翻轉 `DEFAULT_LOCALE` 為 `en`——英文走根路徑 `/`，繁中改走 `/zh-Hant/`。**（採用）**
  - B：加瀏覽器語言偵測 / 重導。否決：靜態站無伺服端；客戶端重導傷 SEO、且與「每語言各自 canonical URL」的既有設計衝突。
- **預設主題**
  - A：`initialTheme()` 回 `light`，移除依系統偏好自動切深色。**（採用）**——CSS 的 `:root` 底色本就是淺色，首屏與 hydration 一致、消除原本淺→深的閃爍；深色仍可手動切換。
  - B：保留「跟隨系統」但預設淺色。否決：使用者明明要淺色，卻可能因 OS 深色而看到深色，違反明確指示。
- **企業版呈現**
  - A：新增獨立頁 `enterprise`（自己的 URL、meta、JSON-LD）。**（採用）**——最可被索引/分享，GEO 友善。
  - B：塞進既有頁的一個區塊。否決：不夠顯眼、無獨立 meta。
- **技術原理擴充範圍**：只收錄**已落地且可公開宣稱**者（分片、多裝置同步、通話 NAT 穿透、換機備援）；**排除 FS**（審計前硬閘）。

## 決策

1. `routes.ts`：`DEFAULT_LOCALE = "en"`。英文走根路徑、繁中走 `/zh-Hant/` 前綴。所有 `routeSlug/routeHref/routeUrl/alternates/sitemap/canonical/x-default` 皆由此推導，自動翻轉。
2. `App.tsx`：`initialTheme()` 回 `light`；移除 `prefers-color-scheme` 自動切換（深色改為純手動）。`theme-color` meta 改為淺色底色。
3. 新增 `pages/Enterprise.tsx` 與 `enterprise` view（納入 `VIEWS`、nav、`seo.pageMeta` 的 `byView`），六張能力卡（封閉自架節點／組織名冊與邀請碼／離職接管無金鑰託管／公司政策／資料主權／開源不鎖定）＋部署說明＋連自架文件 CTA。
4. `pages/Tech.tsx`：新增「底層機制（進階）」區，四張卡＝中繼分片與 presence 分層（0241）、多裝置無衝突同步（0242/0071）、通話與 NAT 穿透（0243）、換機與備援。**不含 FS**。
5. 文案全部雙語進 `copy.ts`（SSOT），並補測試（routes/seo/prerender 反映新預設；Tech/Enterprise SSR 斷言；Tech 明測「不得出現前向保密/forward secrecy」以鎖住 ADR-0245 硬閘）。

## 理由

- 站是 URL 驅動、每 (view, locale) 各有靜態頁；「預設語言」在此就等於「根路徑服務哪個語言」，故翻轉 `DEFAULT_LOCALE` 是最忠實、最少特例的實作，且純函式易測。
- 預設淺色反而**移除**既有 hydration 閃爍（CSS `:root` 本為淺色），是淨改善。
- 企業版獨立頁同時滿足「使用者看得到」與「答案引擎索引得到」，與既有 SEO 架構一致（自動納入 sitemap/hreflang）。
- 技術原理只講「已落地且已可公開」者，維持官網主張的可信度；FS 的排除以測試固定，避免日後不小心宣稱。

## 後果

- 正面：國際受眾第一印象為英文＋淺色；技術原理更完整可信；企業版有正式入口與獨立 SEO/GEO 落點；FS 紅線由測試保護。
- 負面 / 已知殘餘風險：**既有網址語意翻轉**——原繁中根 `/Cinderous/` 現改服務英文，繁中移到 `/Cinderous/zh-Hant/`。對既有外連/索引是一次性位移；因官網屬 pre-release、幾乎無外部連結與既有排名，可接受、不另設 301（靜態站 host 亦無伺服端重導）。
- 後續行動 / 待辦：FS 通過外部審計後（ADR-0245 Phase 3），再評估是否於技術原理新增 FS 卡並移除「不得宣稱」測試；行動版上線後把 Home 的行動版卡加回。
