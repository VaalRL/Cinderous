# 0083. 舊站排水完全內部化（移除所有 UI）

- 狀態：已接受
- 日期：2026-07-12
- 相關文件：ADR-0066（更換中繼站，H3 排水）、ADR-0069（relay 自動搬遷）、ADR-0082（命名詞彙表——本 ADR 取代其中「排水 UI 維持、白話化另議」的暫定）；`apps/desktop/src/App.tsx`、`apps/desktop/src/ui/SettingsPanel.tsx`

## 背景與問題

排水（drain，ADR-0066 H3）機制本來就是**預設開、自動武裝**：任何一次更換中繼站（手動 H2 或自動搬遷 ADR-0069 T2/T3）都會自動記 `previousRelayUrl`＋`drainUntil`，連線池自動多訂一條舊站的自家收件匣（`activeDrain`→`drainUrl`）。沒有「啟用排水」開關。

它唯一露臉的地方是**設定頁的一行「排水中」狀態＋「提前完成排水」按鈕**，加上自動搬遷通知裡的「（排水）」字樣。既然自動更換中繼站對個人身分**預設開**，排水就是它的**必要配套**，「排水」對使用者是難懂的內部術語，露出反而造成困惑（亦是 ADR-0082 盤點出的命名不清點）。

## 決策

**排水完全內部化：保留機制，移除所有使用者可見痕跡。**

- 移除設定頁的排水狀態列與「提前完成排水」按鈕（`SettingsPanel` 的 `drain`/`onDrainComplete` props 一併刪除）。
- 移除 App 層 `completeDrain` 與其 UI 接線（`clearDrain` 匯入一併移除）。
- 自動搬遷通知去除「（排水）」字樣，只保留「已自動更換中繼站、舊站來訊仍會續收 7 天」這則有意義的事件通知。
- 移除三個 i18n 鍵 `settings_relayDrain{,Done,DoneConfirm}`。
- **機制零改動**：`createBackend` 仍以 `activeDrain(p, now)` 決定是否傳 `drainUrl` 給連線池；到期（7 天，對齊 ADR-0065 TTL）自動失效。

## 理由

- **排水是唯讀 + E2E**：只從舊站收自己的密文、不經它發送，中繼只見密文——**隱藏的安全代價很低**，不像關掉某個保護。
- **7 天窗自動閉合**：到期即自動停訂，無需人工「提前完成」。
- **命名問題一併消解**：使用者不再看到「排水」一詞，ADR-0082 關於此詞的白話化爭議自然消失；程式內部續用 `drain`。

## 後果

- 正面：一般使用者對排水完全無感；設定頁更乾淨；`排水` 命名問題消除。
- 負面 / 已知取捨：**放棄「立即斷開舊中繼站」逃生口**——影響兩個少數情境：(a) 不信任舊站營運者想立刻停止連它（僅屬中繼元資料層級，因內容 E2E）、(b) 手機想省掉背景多一條連線。評估後認為代價可接受；未來若有強隱私/行動省電需求，再以「進階」選項補回（另立 ADR）。
- 後續：行動端若日後也接上更換中繼站流程，比照本 ADR——排水純內部、不出 UI。
