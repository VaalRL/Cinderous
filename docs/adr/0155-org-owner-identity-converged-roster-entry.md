# 0155. 企業主身分——組織名冊入口收斂到「新增身分」

- 狀態：已接受（已實作）
- 日期：2026-07-16
- 相關文件：ADR-0145（新增身分選類型——本 ADR 加第三類型）、0047（組織名冊——
  本 ADR 收斂其管理者入口）、0045/0138（多身分登錄）、0046（企業成員邊界）

## 背景與問題

組織名冊（ADR-0047）的管理者工具過去掛在**最外側 idbar 的 🗂 按鈕**，顯示條件是
`activeBackend.publishRoster` 存在——而該方法所有真實 relay 後端都有，等於**每個一般
使用者的頂欄都掛著一顆管理者專用按鈕**。同時「誰是管理者」在模型裡不存在：任何身分
都能發佈名冊，成員填誰的 npub 誰就是信任根——概念成立，但 UI 上沒有可辨識的身分類型。

使用者裁示：一般的新增身分都是一般使用者身分；點「企業名冊」時應**先建立一個一般
身分、再掛上名冊管理功能**，並把這個身分**定義成企業（主）身分**——把管理者入口
收斂進「新增身分」流程。

## 決策

### 1. `Profile.orgOwner?: boolean`——企業主是「一般身分＋名冊管理權」的標記

- 企業主身分的**後端語意與一般個人身分完全相同**（漫遊、錨點、搬家、配對全開；
  `enterprise: false`）。它不是成員（成員的鎖定單座、名冊訂閱是「接受組織管理」的
  結果，管理者不受自己管理）。
- `orgOwner: true` 只決定兩件事：身分切換器前綴顯示 🗂，以及 idbar 名冊按鈕是否顯示。

### 2. 新增身分第三類型：🗂 企業主（ADR-0145 的兩步流程加一顆）

`AddIdentityModal` 的選類型步驟變三顆：👤 個人／🏢 企業成員／🗂 企業主。企業主表單
＝個人表單（無「管理者 npub」欄——**自己就是管理者**）；建立時寫入 `orgOwner: true`。

### 3. 建立後首次進入自動開啟名冊管理

「點企業名冊＝建身分＋名冊功能」的體驗：建立企業主身分後首次進入（Tauri 走
`location.reload()`，以 `nb.rosterIntro.<pubkey>` localStorage 旗標跨越重載；瀏覽器
原地切換直接開）自動彈出 `RosterAdminModal`，管理者立刻能填成員、發佈名冊，並把
自己的 npub 交給成員（成員在 🏢 企業成員類型填入）。

### 4. idbar 🗂 只對企業主顯示

顯示條件從 `activeBackend.publishRoster`（人人都有）收緊為
`publishRoster && activeProfile.orgOwner`。一般與成員身分的頂欄不再出現管理者按鈕。
順手把 idbar 三顆按鈕的硬編碼中文 tooltip（新增身分／解鎖隱藏身分／組織名冊）補進
i18n（`idbar_addIdentity/unlockHidden/roster`）。

## 兩種企業相關身分的分工（本次盤點結論，記錄備查）

| | 🏢 企業成員（ADR-0046/0047） | 🗂 企業主（本 ADR） |
|---|---|---|
| 建立 | 新金鑰＋`enterprise: true`＋可選 `adminPubkey` | 新金鑰＋`orgOwner: true`（其餘同個人） |
| relay | 鎖定單座：無 connectorFor/anchors/清單學習/自動搬家；設定內鎖定不可換 | 與個人相同（漫遊/遞補/搬家全開） |
| 名冊 | **訂閱**：驗簽採用 → 自動通訊錄、組織群組對帳、身分輪替、政策（forceTurn、禁雲備份） | **發佈**：簽章名冊、匯出 allowlist |
| 配對搬家 | 不提供（ADR-0072 v1 排除） | 提供 |

## 後果

- 正面：管理者入口成為一級身分概念、可被發現（ADR-0145 的初衷延伸）；一般使用者
  頂欄不再有無關按鈕；「誰是管理者」在 UI 與模型中可辨識。
- 已知限制／取捨：
  - **既有**曾以一般身分發佈過名冊的管理者，升級後看不到 🗂（該身分無 orgOwner
    標記）；因名冊信任根＝金鑰，換新企業主身分＝成員須改填新 npub。目前無生產部署，
    接受；日後如需可加「把既有身分標記為企業主」的設定入口（新 ADR）。
  - 行動端維持既有單一表單（同 ADR-0145 的既有限制），亦無名冊管理 UI；待後續對齊。
  - 舊 Profile 無 `orgOwner` 欄位 → 讀取視為 false，零遷移。
- 測試：AddIdentityModal 選類型三顆／企業主表單無管理者欄／類型標示；既有測試不變。
