# 0037. 文字觸發貼圖：composer 尾端比對 + 建議列（Tab 送出）

- 狀態：已接受
- 日期：2026-07-02
- 相關文件：docs/adr/0021/0032（貼圖）、apps/desktop/src/ui/sticker-prefs.ts、
  apps/desktop/src/ui/sticker-triggers.ts

## 背景與問題

使用者希望為貼圖設定觸發文字：輸入觸發文字時，能以快捷鍵或滑鼠快速
送出對應貼圖。訊息格式是「整則即貼圖」（`nb-sticker:v1/v2`），無法與
文字混排，「插入」的語意需要定義。

## 決策

- **資料模型**：`{ trigger → StickerRef }` 對照表（重用既有參照系統：
  內建 `{pack,id}`、自製 `{pack:"__custom", id:內容雜湊}`），localStorage
  `nb.stickers.triggers`，純函式運算 + 薄包裝（同 sticker-prefs 模式）。
  約束：trim 後 1–16 字、拉丁字母不分大小寫、一個觸發字對應一張貼圖
  （覆蓋需確認）、一張貼圖可多個觸發字、總數上限 64。懸空參照（貼圖
  已刪）沿用最近/最愛行為：解析不到即不顯示，不做級聯清理。
- **比對規則**（`matchTriggers`，純函式）：composer 文字**尾端**等於
  某觸發字的前綴且**長度 ≥ 2** 即建議（中文無空白分隔，不能以詞切；
  ≥2 防止任意字元都彈窗）；單字觸發字需完整命中。排序：完整命中優先、
  命中長度降冪、觸發字字典序；上限 5 筆。
- **接受語意**：接受＝**立即送出該貼圖訊息**，並從 composer 尾端剝離
  命中片段（其餘文字保留不送）、記入最近使用。
- **鍵盤**：**Tab** 接受選中（預設第一）、**↑/↓** 移動、**Esc** 關閉至
  下次輸入變化。**不劫持 Enter**——Enter 維持送出文字，避免誤送貼圖
  的破壞性誤觸。
- **設定入口**：貼圖格 hover 左上 ⌨，`window.prompt` 編輯該貼圖的
  觸發字清單（空白分隔、清空即全移除）；佔用衝突逐一確認覆蓋。
- 觸發字為純本地設定，不隨訊息外流。

## 理由

- 尾端前綴比對是中英文通用的最小規則，純函式可完整測試。
- 「接受即送出」與訊息格式（整則即貼圖）一致，避免發明混排格式。
- Tab 而非 Enter：送出貼圖是不可逆動作，不能掛在最高頻按鍵上。

## 後果

- 正面：零協定變更、零新依賴；自製貼圖觸發後照走 v2 內容隨訊息。
- 負面：≥2 規則下單字英文觸發需打完整字；建議列不支援模糊比對。
- 後續：觸發字總覽管理面板；per-字首索引（觸發字數量大時）。
