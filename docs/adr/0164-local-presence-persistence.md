# 0164. 本機記住上線狀態與自訂狀態文字——下次上線還原（中繼站仍 Ephemeral）

- 狀態：已接受（已實作）
- 日期：2026-07-17
- 相關文件：ADR-0129（狀態封裝廣播——本 ADR 不動其傳輸/Ephemeral 性質）、
  0088（隱身）、0142（自訂狀態文字）；全域鐵則「中繼站不持久化線上狀態」

## 背景與問題

盤點發現：自訂狀態文字、上線狀態（online/away/busy/offline）、正在聽的音樂全部**只在
記憶體**——後端建構時一律 `status:"online"、statusMessage:""`，開機不從儲存讀回，也無任何
localStorage 鍵。結果每次重載/重啟/切身分/重新登入都重置，使用者每次上線要重打自訂狀態。

「中繼站不持久化」（隱私鐵則）與「本機不記住」是兩件事：狀態在**中繼站**不落地是對的，
但在**自己的裝置**本機記住、下次上線自動套用，不牴觸任何隱私性質（同頭像/暱稱/通知偏好）。

## 決策

1. **本機依身分持久化**：`nb.presence.<pubkey>` 存 `{status, statusMessage}`（純本機、
   不進 Nostr、不進雲端快照）。純函式 `loadPresence`/`savePresence`（形狀防禦，壞值退回）。
2. **只記手動選擇**：使用者改狀態/狀態文字（App 的 `setStatus`/`setStatusMessage`）時寫入。
   **閒置自動 away 不落地**——它走另一條 `apply` 路徑（不經這兩個 handler），故隔天上線
   還原的是「你上次**手動**設成什麼」，而非離開電腦前被自動改成的 away。
3. **上線還原**：建立後端時把記住的值塞進 `self`（讓稍後的閒置初始化以此為基準，避免
   idle 把狀態重置回 online）；`start()` 後再 `backend.setStatus(記住值)` 同步後端並廣播
   （online 時 beat＋廣播；`offline` 靜默——與「上次把自己設為離線」一致）。
4. **正在聽的音樂不持久化**：`nowPlaying` 語意是「當下」正在聽，不該黏著跨 session；維持
   Ephemeral。

## 理由

- 本機記憶純屬使用者裝置偏好，與「明文/狀態不上雲」鐵則正交——中繼站看到的仍是每次
   連線後即時廣播的 Ephemeral 狀態，不多一分持久化。
- 只記手動、閒置不落地：語意最符合直覺（還原你的選擇，不還原你離開時的自動狀態）。

## 後果

- 正面：上線即回到上次的狀態與自訂狀態文字，免重打。
- 已知限制／取捨：
  - 「上次設離線」會在下次上線還原為離線（appear offline）——faithful，但使用者若忘記
    自己設過離線，可能一時不解為何聯絡人看不到自己（狀態選單仍清楚顯示為離線）。
  - 依 pubkey 命名空間：多身分各記各的；`nb.presence.<pubkey>` 目前不隨換機/搬家同步
    （純裝置偏好，與 chatbg/avatar 一致）。
  - 行動端未接（桌面先行，同既有慣例）。
- 測試：desktop（presence-store round-trip、壞值退回、只認合法 status；依身分隔離）。
