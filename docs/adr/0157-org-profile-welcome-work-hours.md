# 0157. 公司設定隨名冊分發——歡迎詞／表定上下班時間＋下班自動靜音

- 狀態：已接受（已實作）
- 日期：2026-07-16
- 相關文件：ADR-0047（組織名冊——本 ADR 擴充其文件格式）、0155/0156（企業主與入職
  自動化——公司設定由企業主在同一名冊管理視窗維護）、0076（通知子設定）

## 背景與問題

企業主（ADR-0155/0156）能佈建成員與政策，但「公司本身」的設定只有組織名稱。使用者
要求：公司名稱、基本規範（新成員加入時的預設公告/歡迎詞）、公司表定上下班時間都可
設定。上下班時間的行為效果經使用者裁示：**顯示＋下班時間自動靜音組織通知**。

## 決策

### 1. 名冊文件擴欄位（ADR-0047 格式向後相容）

`OrgRosterDoc` 加兩個可選欄位，隨管理者簽章分發、成員採用時生效：

- `welcome?: string`——歡迎詞／基本規範（≤2000 字，sign/verify 皆修剪與上限防禦）。
- `workHours?: { start, end }`——`"HH:MM"` 24 小時制；`start === end` 或格式不符視為未設。

`verifyOrgRoster` 本就以「已知欄位重建」方式寬鬆解析——舊版客戶端收到新欄位自動忽略，
零協定版本切換。核心新增純函式 `inWorkHours(wh, minutesOfDay)`，**支援跨夜班表**
（如 22:00–06:00）。

### 2. 成員端：組織資訊＋一次性歡迎詞＋下班靜音

- 引擎 `adoptRoster` 發出 `onOrgInfo { org, members, welcome?, workHours? }`（members＝
  在世成員 pubkey，供靜音判定）。
- **歡迎詞**：成員端以 `nb.orgWelcome.<pubkey>` 記住上次顯示過的內容——首次採用或
  **內容變更時**一次性彈窗（標題＝公司名稱），平時可在設定 → 組織資訊隨時查看。
- **下班自動靜音**（使用者選項）：視窗未聚焦的桌面通知在表定時間外、且來源是
  **組織成員（1:1）或組織群組**時，不彈通知、不響音效；**未讀徽章照常**（靜音不是
  漏訊）。非組織聯絡人不受影響。判定為純函式 `shouldMuteOrgNotification`（可測）。
- 設定面板新增「組織資訊」區（唯讀）：公司名稱、規範/歡迎詞、上下班時間與靜音說明。

### 3. 企業主端：同一名冊視窗維護＋既有名冊預填

- 名冊管理視窗新增「歡迎詞／基本規範」textarea 與上下班 `time` 輸入。
- 引擎新增 `currentRoster()` getter：視窗開啟時以現行名冊**預填**組織名稱、成員清單
  （npub＋名稱行）、政策勾選、歡迎詞、上下班——修一項設定不再需要重打整份名冊
  （企業主重啟後由 ADR-0156 的自我訂閱找回）。
- ADR-0156 的自動核准重發（`approveJoin`/`pendingJoins`）**保留** welcome/workHours
  ——新人入職不會把公司設定洗掉。

## 理由

- 名冊本來就是「管理者簽章的公司狀態」單一真實來源（ADR-0047）——公司設定放進同一
  份文件，分發、防清空、較新才取代、重啟找回全部免費繼承。
- 靜音只擋「打擾面」（彈窗/音效），未讀與訊息本體不受影響——下班保護不變成漏訊風險。

## 後果

- 正面：公司名稱/規範/班表集中設定、全員自動同步；下班時間組織訊息不再打擾；
  名冊視窗從「每次重打」變成「載入現況改一項」。
- 已知限制／取捨：
  - 班表全公司一份（無分部門/時區；成員本地時鐘判定）。多班表/例外日為後續。
  - 靜音以「名冊成員＋組織群組」界定來源；組織成員在**非組織群組**發言不靜音（刻意，
    那是私人社交面）。
  - 群組欄位（組織群組文字格式）尚未預填（id 重建複雜）；歡迎詞變更即再彈一次
    （視為新公告，設計如此）。行動端未接（同 0155/0156）。
- 測試：core（新欄位 round-trip/防禦、inWorkHours 跨夜/邊界）＋engine（onOrgInfo、
  approveJoin 保留設定、currentRoster）＋desktop（靜音判定、視窗欄位/預填、組織資訊區）。
