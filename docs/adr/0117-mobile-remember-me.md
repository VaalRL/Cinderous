# 0117. 行動端「記住我」：以本地密碼包裹 nsec

- 狀態：已接受（已實作）
- 日期：2026-07-14
- 相關文件：ADR-0053（OS 金鑰庫）、0067（本地密碼）、0073（忘記密碼救援）、
  0081（行動端登入）、**0112（web/mobile 靜態加密：nsec 不得明文落盤）**

## 背景與問題

ADR-0112 定下紅線：**web/mobile 一律不明文儲存 nsec**。行動端因此**每次開 App 都要重貼一次
nsec**。

那不只是不方便——它**製造了一個更糟的行為**：使用者會把 nsec 貼到記事本、雲端筆記、
聊天視窗裡，好方便下次複製。**我們把私鑰從 localStorage 趕出去，結果它跑到雲端筆記裡。**

而解法早就寫好了：ADR-0112 已經實作了 `wrapSecret` / `unwrapSecret`
（Argon2id KEK，參數與桌面 `passlock.rs` 一致）。**只缺 UI 與流程。**

## 決策

### 1. 「記住我」＝以 Argon2id 密碼包裹 nsec 落地

登入畫面加一個**選填**的本地密碼欄。有填 → 包裹後存 localStorage；沒填 → 不落地
（nsec 只留在記憶體，重開要重貼——即現況）。

磁碟上只有密文；KEK 由密碼導出、**從不落盤**。

### 2. **絕不無密碼記住**

`rememberIdentity(identity, "")` 回 `null`。無密碼的「記住」＝明文存 nsec ＝ 違反 ADR-0112。
這是**強制點**：呼叫端就算想，也存不進去。已有測試釘住。

### 3. **只認 Argon2id 包裹的 blob**

`isRemembered()` 會檢查 `wrapped` 確實是 `{ v:1, kdf:"argon2id", … }`。
有人（或某個未來的 bug）往那個鍵塞明文 nsec，**一律不收**——當作沒有記住的身分。

### 4. 忘記密碼＝**沒有救援**，但有出路

密碼是**唯一**的 KEK 來源。桌面的 ADR-0073 救援靠的是**nsec 本身**解開第二道包裹——
而這裡的前提正是「使用者手上只有密碼、沒有 nsec」，那條路不成立。

所以解鎖畫面提供**「改用 nsec 登入」**：重貼 nsec 即可，而且**記住的資料不會遺失**
（同一個 pubkey ＝ 同一個 localStorage namespace ＝ 同一把 DEK，ADR-0112）。
另提供「忘記這個身分」清掉密文。

### 5. 登出＝連「記住的身分」一起清

否則登出後下次開機又被帶回解鎖畫面——那不叫登出。

## 理由

- 這**不是「假安全感」**（ADR-0112 已推翻 ADR-0067 的這個推論）：它擋不住頁面內的惡意 JS
  （桌面的 webview 同樣擋不住），但它**擋住「有人拿到這台裝置／複製了 localStorage」**。
- 而**不做的後果不是誠實，是使用者把 nsec 貼到雲端筆記裡**——那才是真正的安全災難。
- 密碼錯誤與遭竄改**回同一個錯誤**：不給攻擊者可用的訊號。

## 後果

- 正面：
  - 行動端可以安全地「記住我」——磁碟上只有密文。
  - 與桌面**同一套 Argon2id 參數與 blob 格式**（ADR-0112）——不留平行實作。
  - 測試 +7（往返、**落地不含明文 nsec**、**絕不無密碼記住**、密碼錯誤、竄改、
    **只認 Argon2id blob**、鹽隨機）。全 repo **934 通過**。

- 已知限制：
  - **忘記密碼無法救回**（見決策 4；有「改用 nsec」的出路）。
  - **擋不住頁面內的惡意 JS**（webview 架構的固有邊界，非本 ADR 引入）。
  - 未提供「改密碼」／「停用密碼」——目前只有「記住」與「忘記」。
  - 行動端仍缺：多身分、貼圖、相簿、AI、音樂狀態、多中繼管理、配對搬家（送出端）。
