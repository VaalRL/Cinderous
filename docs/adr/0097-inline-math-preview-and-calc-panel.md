# 0097. 對話算式預覽：composer 即時預覽＋右欄計算機（純本地、禁用 eval）

- 狀態：已接受（已實作）
- 日期：2026-07-13
- 相關文件：ADR-0060（本機 AI 改寫）、0079（三欄佈局與右側輔助欄）、0085（行動端）；
  `packages/core/src/calc.ts`、`apps/desktop/src/ui/{ConversationWindow,DeckRight}.tsx`

## 背景與問題

使用者希望在對話中輸入數學算式時能看到答案（LINE 有類似功能）。需求延伸為：**右側輔助欄有自己的
輸入框**，在那裡計算，再**自行決定是否插回主對話框**——不要偷改正在打的訊息。

## 考量的選項

- **A. 只做 composer 即時預覽**（LINE 式）：全平台可用、就在眼睛所在處；但會在打字時彈出提示，
  有誤判擾民風險，且無法「先算再決定要不要用」。
- **B. 只做右欄計算機**：不打擾打字、可先算再插入；但**右欄只存在於三欄（modern）佈局**——
  經典佈局與行動端完全沒有。
- **C. 兩者都做（採用）**：互補。右欄＝「先算再決定」的草稿計算機（三欄）；composer chip＝
  「正在打算式」的即時預覽（全平台，含經典佈局與行動端）。兩者共用同一個純函式核心。

## 決策

### 1. 核心：`packages/core/src/calc.ts`（純函式，桌面/行動共用）

- **安全紅線：絕不使用 `eval` / `new Function`**。這是加密通訊 App，把使用者輸入丟進 JS 執行器
  等同開一個任意程式碼執行面。改用自寫 **tokenizer + 遞迴下降 parser**，只吃白名單文法：
  數字、`+ - * / % ^`、括號、一元正負號。硬上限：輸入 64 字元、括號深度 32（防病態輸入/爆堆疊）。
- **全形正規化**：`＋－×÷（）％＾`、全形數字、`−`(U+2212) → 半形（繁中輸入法會打出全形）。
- **保守的觸發條件**（誤判擾民比漏判更糟）：
  - **整串** trim 後必須完全符合文法——**不抓子字串**（「我 1+1 對嗎」不觸發）
  - 至少一個運算子（純數字 `42` 不觸發）
  - 結果必須為有限數（`1/0` 不顯示）
  - 排除**日期/電話**樣式：三段、同一分隔符，且有補零段或首段像年份
    （`2024/01/02`、`2024/1/2`、`02-1234-5678` 擋掉；但 `10-2-3`、`100/4/5` 這種正常算式**仍可計算**）。
    版號 `1.2.3` 由斷詞階段自然拒絕（一個數字裡兩個小數點）。
- **浮點毛邊**：以 `toPrecision(12)` 抹除（`0.1+0.2` → `0.3`）。

### 2. 右欄計算機（`DeckRight` 新增「計算」分頁）

- **自己的輸入框**，完全不碰主對話框草稿——所以**不需要把 composer 的草稿狀態上提**。
- 顯示「算式 = 答案」，提供兩顆按鈕：**插入算式與答案** / **只插入答案**。

### 3. 插入通道：單向指令，不接管草稿

App 持有 `pendingInsert: { convo, text, nonce }`；`ConversationWindow` 以 `useEffect` 監看 **nonce
變化**，把文字**附加**到自己的草稿。刻意**不**把草稿狀態上提到 App——那會牽動 @提及建議、串內
composer 等既有狀態機，風險遠大於效益。

### 4. composer 即時預覽 chip（桌面＋行動端）

草稿是算式時，composer 上方出現 `= 答案` chip，點一下把「= 答案」附加到草稿。

## 理由

- **隱私零成本**：算式在**草稿階段**就地計算——草稿根本還沒送出、不進中繼、不上網、**不碰 AI**。
  與 ADR-0060 的 AI 改寫（會把文字送去 Ollama/OpenAI）性質完全不同，這裡是純本地純函式。
- **不動協定**：沒有新 event kind、不碰 relay、不碰加密。
- 純函式核心 → 高度可測（優先序、括號、右結合次方、全形、除零、日期不誤判、JS 片段不觸發…）。

## 後果

- 正面：全平台都有即時預覽；三欄另有「先算再決定插不插」的草稿計算機；核心可測且零隱私成本。
- 負面 / 已知殘餘風險：
  - **右欄計算機只在三欄（modern）佈局**——經典佈局與行動端只有 composer chip（設計如此，非缺陷）。
  - 觸發啟發式必然有邊界情況：`2024/1/2` 被當日期擋掉（不算 1012）；`1/2/2024` 這類美式日期
    目前仍會被算成小數。取捨傾向「寧可漏判、不要誤判」。
  - `%` 解為**模數**（非百分比）；`20%` 這種寫法不會給出 0.2（語法不完整 → 不顯示，不會給錯答案）。
  - 目前無開關可關閉預覽；若實際使用覺得吵，再加設定（尚無證據支持先做）。
- 測試：core `calc.test.ts` 11 項（含「JS 片段一律不觸發」作為禁用 eval 的行為證明）；
  desktop DeckRight 計算分頁結構測試。全 755 測試通過。
