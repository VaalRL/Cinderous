# 0238. 輪替加密子鑰前向保密：設計與規格

- 狀態：已接受（ADR-0236 選項 3 的實作設計＝前向保密的實作依據；prototype／實作待排、**上線前需外部審計**）
- 日期：2026-07-23
- 相關文件：**ADR-0236**（前向保密：選定「輪替加密子鑰」路徑，本 ADR 是其設計）、
  ADR-0002（隱私元資料基線）、ADR-0034（多中繼路由）、ADR-0035（加密 rumor 學 relay hint）、
  ADR-0057（NIP-42 AUTH）、ADR-0061（加密個人檔）、ADR-0066（開機廣播個人檔）、
  ADR-0071（加密雲端快照）、ADR-0072（配對克隆）、ADR-0107（多設備自封同步）、
  ADR-0160（企業 TTL）、ADR-0237（元資料可連結性）、`PRD.md §4／§6`

## 背景與問題

ADR-0236 選定以「輪替簽章加密子鑰」取得**粗粒度前向保密**（給 FS、不給 PCS，避開 per-message
棘輪的多設備地雷）。本 ADR 把它從方向落成**可實作的規格**，並定案先前列出的開放決策。

回顧機制：身分金鑰 IK（npub）不動；另生一把**加密子鑰** `EK_e`（每 epoch 一把、每身分一把、
**隨機**非 nsec 導出），其公鑰由 IK 簽章後發佈。寄件人以**一次性臨時金鑰**對收件人**當前 EK**
加密（retarget 現有 Gift Wrap）；收件人在 epoch＋grace 後於**所有裝置與快照**刪除 `priv(EK_e)`
→ 事後即使 nsec 失竊，也解不開被側錄的網路密文。**這是傳輸/E2E 金鑰，與本機 at-rest 儲存金鑰
（nsec 導出、不變）是兩回事**：訊息到達時用 `priv(EK_e)` 解一次網路密文、存進本機封存；日後讀
舊訊息走本機封存（at-rest 金鑰），不再碰 EK。故刪 EK 得 FS、又不弄丟本機歷史。

## 決策

### 1. Epoch 與金鑰壽命

- **epoch＝固定 UTC 週**：`epoch = floor(unix_time / 604800)`。雙方看時鐘即算出同一週數、無需協商。
- **輪換週期＝每週**（已定）。
- **grace＝固定 7 天，且獨立於企業 TTL**（已定）。一把 `EK_e` 最長活 epoch(7d)＋grace(7d)＝**14 天**。
  - 🔴 **不隨 `MAX_TTL_DAYS`（ADR-0160，可達 3650 天）放大**：否則 EK 活好幾年、FS 失效。代價：
    離線超過 grace 才上線者，那批訊息的網路密文可能已無 EK 可解（超過 grace 的離線訊息接受可能
    收不到）。grace 為全域上限、不可被站方 TTL 覆寫。

### 2. `EK_e` 生成、多裝置與衝突解決

- **隨機、每身分一把**：`EK_e` 是獨立隨機 secp256k1 金鑰對（**不可**從 nsec 導出，否則無 FS）；
  一把服務所有聯絡人（非每對話一把，避免 N×）。
- **生成＝自由生成 ＋ 可取代事件 LWW 權威 ＋ 全私鑰同步**（取代「選 leader」，免單點/開窗）：
  - 任一裝置在 epoch 滾動時可生成並發佈 `pub(EK_e)`（見 §3）；發佈為**可取代事件**，NIP-01 LWW
    （較新 `created_at` 勝、id 字典序決勝）在中繼收斂出**唯一權威公鑰**。寄件人只用這把權威公鑰。
  - **所有生成過的 `EK_e` 私鑰（含 race 中落敗的）都經 §4 同步到每台裝置，並留到 grace。** 於是
    不論寄件人當下看到/用了哪一把，收方任一裝置都解得開。
  - **殘餘**：某訊息用的 EK 私鑰尚未同步到「當下嘗試解密的那台裝置」→ 該訊息暫時顯示未解，
    待同步後重試（所有多裝置 E2E 的通性，非本設計獨有）。

### 3. 發佈與發現（子鑰 PKI）＝**混合**（已定）

- **公開可取代事件（discovery）**：新增 **kind 10040**（`EK 公告`，接續 10037/10038/10039），
  由 IK 簽章，內容帶**當前**與**下一個** epoch 的 `pub(EK)`（**預發下一週**以消除週界空窗）。
  可取代＝每身分一顆最新。中繼可見「此 npub 每週換加密子鑰」——極小元資料（ADR-0237 已知、接受）。
- **加密 rumor 內嵌（zero-metadata 即時更新）**：你送出的每則訊息，在**加密內層 rumor** 帶一份
  當前 `pub(EK)`（比照 ADR-0035 的 relay hint），對方從對話中即時學到、無需額外訂閱。
- 兩者並用：公開事件解「首次接觸（還沒收過你訊息）」，rumor 內嵌解「即時、免元資料」。

### 4. 多裝置同步與新裝置上線

- **EK 私鑰同步**：走既有 ADR-0107 自封 NIP-17 同步事件；併入 ADR-0071 加密雲端快照；
  併入 ADR-0072 配對克隆包。
- **新裝置上線帶「當前 ＋ grace 內的舊 EK 私鑰」**（已定）：足以解在途與 grace 內的訊息；不帶
  更舊的（已刪）。

### 5. 加密改動（retarget Gift Wrap）

- **外層 wrap 加密到收件人當前 `EK_e`**（而非收件人靜態金鑰）：wrap 一旦具 FS，其內 seal 亦無法
  觸及。**seal 亦一併 retarget**（縱深防禦）。外層 `#p` **仍為收件人真名 npub**（供中繼路由/收件匣）。
- **sender 側 FS**：沿用 Gift Wrap 既有的一次性臨時 wrap 金鑰（用後即刪）→ 送出者日後 nsec 失竊
  也解不開自己送出的網路密文。seal 仍由 sender IK 簽章（**認證不變**），但加密對手方為收件人 EK。

### 6. 降級保護（capability 宣告 ＋ 偵測）（已定要設計）

- **簽章 capability 旗標**：使用者的**加密個人檔**（ADR-0061/0066）帶 `fs: "ek-v1"`。個人檔由 IK
  簽章 → MITM **無法偽造「不支援 FS」**；唯一手段是丟棄整份個人檔，而那對已知聯絡人是可疑訊號。
- **TOFU 釘選**：對方首次宣告 `fs: ek-v1` 即**本機釘選**「此聯絡人需 FS」。
- **寄件策略**：
  - 對**已釘選 FS** 的聯絡人，若**找不到可用 EK** → **不得靜默退回靜態金鑰**；先試「上一週 EK」與
    「近期訊息內嵌的 EK」排除正常空窗，仍無 → **警告/暫緩/重試**（區分「暫時無法建立 FS、稍後重試」
    與「疑似降級」）。
  - 對**未釘選**（舊版/未升級）聯絡人 → 允許靜態退回（無 FS，但屬合法）。
- **殘餘**：首次接觸前若旗標被剝除，使用者從未釘選 → TOFU 首用弱點（所有 TOFU 皆有），日後以
  帶外驗證（安全碼）緩解，屬後續。

### 7. 刪除紀律

- epoch＋grace 後，於**所有裝置**刪 `priv(EK_e)`，並**從快照/配對包老化剔除**（否則舊快照外流即
  破 FS）。備份碼/救援（ADR-0070/0073）還原身分時**不還原**已過 grace 的 EK。
- **安全刪除**為 best-effort（localStorage/OPFS/SSD/keychain 不保證抹除位元組）→ 記為已知殘餘。

### 8. 範圍與相容

- **僅 1:1**；**群組不納入 v1**（可套「輪替群組金鑰」拿粗粒度群組 FS，但被踢成員 PCS 仍需
  rekey＝MLS，另議）。
- **不追溯**：既有靜態金鑰加密的訊息維持原樣、拿不到 FS；只有新訊息享 FS。
- **向後相容退回**：對方無 EK 且未釘選 FS → 退回現行靜態 Gift Wrap。

## 理由

- **為何生成走「自由生成＋LWW＋全私鑰同步」而非選 leader**：leader 制有單點與「leader 離線即
  開天窗」的問題；LWW＋全私鑰同步讓「寄件人用了哪把」與「收方能否解」解耦——只要私鑰終會同步到
  每台，任何一把都解得開，代價僅是 race 期間多同步一兩把（grace 後自然老化）。
- **為何降級用「簽章旗標＋TOFU 釘選」**：這是 SSH host key／Signal safety number 的成熟範式。旗標
  在**簽章**個人檔內＝不可偽造，剝除整份個人檔對已知聯絡人可疑；釘選把「靜默降級」變成「可偵測的
  異常」。首用 TOFU 弱點以帶外驗證另補。
- **為何 grace 固定、不隨企業 TTL**：FS 的價值就在「金鑰壽命有界」；讓站方 TTL 把 EK 壽命拉到數年
  等於自廢 FS。寧可犧牲「離線超過 grace 的極少數訊息」。
- **為何 seal 也 retarget（雖 wrap 已足）**：縱深防禦，成本近乎零（同一次加密改對手方）。
- **為何 kind 10040 公開可取代**：discovery 需要一個「首次接觸也找得到」的錨；可取代＝零累積；
  元資料極小且 ADR-0237 已接受「輪替訊號」這類殘餘。rumor 內嵌則覆蓋即時/免元資料路徑。

## 後果

- 正面：
  - 1:1 訊息取得**粗粒度前向保密**（粒度 ≈1～2 週）：金鑰刪除後，被側錄的網路密文在 nsec 日後失竊
    時仍解不開；且**不弄丟本機歷史**（走 at-rest 封存）。
  - **避開 per-message 棘輪的多設備地雷**（無序列鏈、無 counter 碰撞、無跳過金鑰窗 DoS）。
  - 改動局部（retarget 現有 Gift Wrap＋一顆可取代事件＋rumor 內嵌），複用既有同步/快照/配對。
  - 降級保護把「靜默退回靜態」變成可偵測異常。
- 負面 / 已知殘餘風險：
  - **無 PCS**；FS 粒度為 epoch＋grace。
  - **離線超過 grace（14 天）的訊息**其網路密文可能已無 EK 可解（grace 不隨企業 TTL 放大的取捨）。
  - **race 期間「EK 私鑰未及同步」**→ 該訊息暫時未解、待同步重試。
  - **TOFU 首用弱點**（旗標首次接觸前被剝除）。
  - **安全刪除 best-effort**（存媒殘留）。
  - **本機明文封存仍是最大長期暴露面**（FS 本就管不到；靠 at-rest 加密＋限時訊息）。
  - **kind 10040 洩漏「此 npub 使用 FS 輪替」**的極小元資料（ADR-0237 接受）。
- 後續行動 / 待辦：
  1. **Prototype**：`packages/core` 新增 `subkey.ts`（EK 生成/簽章/公告事件建構與驗證、當前/下一 epoch
     計算，純函式可測）；retarget `nip59.sealAndWrap`／`giftwrap` 到收件人 EK；`giftwrap` 解封時
     依序嘗試「當前→上一週→rumor 內嵌」EK。
  2. 引擎接線：EK 生成排程（週界）、kind 10040 發佈/訂閱、rumor 內嵌與學習、多裝置同步（ADR-0107）、
     快照/配對老化剔除（ADR-0071/0072）、降級偵測與 UI 警告。
  3. TDD 測向量：刪 EK 後網路密文不可解、但本機封存可讀；LWW 收斂與全私鑰同步；grace 邊界；
     降級偵測（旗標剝除→警告，非靜默退回）；向後相容退回。
  4. **外部密碼學審計**後才上線；上線前文案不得宣稱 FS（ADR-0236）。
