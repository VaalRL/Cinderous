# 0116. 行動端通知——並把 Web Notification 基質收斂為單一來源

- 狀態：已接受（已實作）
- 日期：2026-07-14
- 相關文件：ADR-0076（桌面通知）、0088（在線元數據最小化）、0106（webview 節流實測）、
  0113（本地啟動器）、0114/0115（行動端對齊一、二）

## 背景與問題

行動端**完全沒有通知**——App 切到背景就等於「什麼都不知道」。對即時通訊而言這是最基本的一環。

而桌面的 `browserNotifier`（`apps/desktop/src/native/notify.ts`）是**純 Web Notification API**，
一行 Tauri 相依都沒有——**正是行動端需要的東西**。照抄一份到 `apps/mobile` 會是典型的
平行實作（Fix First 明令禁止）。

## 決策

### 1. 把 Web Notification 基質抽到 `@cinder/engine`（單一來源）

`webNotifier`（權限＋送出）、`onWebNotificationClick`、以及**內文組裝的純函式**
`notificationFor()` 移入 `packages/engine/src/notify.ts`。

- 桌面 `native/notify.ts`：保留 Tauri 原生 toast 分支，瀏覽器路徑改用共用基質。
- 行動 `native/notify.ts`：平台縫，直接沿用共用基質。

**內文組裝抽成純函式是重點**：它是「隱藏預覽」這個隱私開關的實作處，必須可測——
而兩端 UI 都用 SSR 渲染測試（不跑 effect），邏輯不抽出來就測不到。

### 2. 何時跳通知：**他人訊息 ＋ App 在背景**

正在看那個對話就別打擾（`viewing` 判斷與未讀水位共用同一個條件）。

### 3. 權限必須在**使用者手勢**裡請求

瀏覽器會拒絕非手勢的 `Notification.requestPermission()`。故：開關按下時才請求。
**使用者拒絕授權 → 開關不打開**（不假裝成功——那會讓人以為通知壞了）。

### 4. 「隱藏預覽」是隱私開關，不是偏好

通知會出現在**鎖定畫面／通知中心**——那是裝置的**非加密表面**。整個專案費盡力氣做
E2E 加密與靜態加密（ADR-0054/0112），若通知把明文推到鎖定畫面，等於在最後一哩自己開洞。

開啟時**只說「有新訊息」**，連群訊的發送者名都不放。已有測試釘住。

（通知關閉時不顯示這個開關——沒有通知就沒有預覽可隱藏。）

## 不做什麼（明確劃線）

**不做推播（FCM / APNs）。**

Web Notification 與 expo-notifications 都是**本機通知**——**只在 App 還活著時有效**。
真正的「App 關掉也收得到」需要推播服務，而那會把「**這個人在這個時間收到訊息**」洩漏給
Google／Apple——**與 Cinder 的威脅模型直接衝突**（ADR-0088 才剛把在線元數據最小化）。

若日後要做，必須另立 ADR，並認真處理：推播內容一律不含明文（只送「有新訊息」）、
且中繼站要能在不知道收件人是誰的前提下觸發推播（技術上很難，可能根本做不到）。

**現況的誠實說法**：行動端（RN-web）通知**只在瀏覽器分頁還開著時**有效。
而分頁在背景時**計時器會被節流**（ADR-0113 已記）——收訊是網路事件、不受影響，
所以通知仍會跳；但送訊會延遲。

## 後果

- 正面：
  - 行動端有通知了（開關 ＋ 隱藏預覽 ＋ 點擊回跳到該對話）。
  - Web Notification 基質**收斂為單一來源**——桌面的瀏覽器路徑與行動端用的是同一份實作。
  - 內文組裝變成可測的純函式；桌面既有的 7 個通知測試**未動一行**仍全過。
  - 測試 +8（engine 5：1:1／群訊前綴／**隱藏預覽絕不放內容**／隱藏預覽也不放發送者名／
    convo 一定帶著；行動 3：示範模式不顯示、通知關閉時不顯示隱藏預覽、開啟時才顯示）。
    全 repo **927 通過**。

- 已知限制：
  - **App 關掉就收不到**（本機通知的本質；見「不做什麼」）。
  - 行動端仍缺：多身分、本地密碼、貼圖、相簿、AI、音樂狀態、多中繼管理、配對搬家（送出端）。
  - 桌面的「通知音效」（ADR-0076）未移植到行動端。

- 一個 i18n 教訓（第二次了）：`settings_notifications` **早就存在**，我第一版又新增了
  `settings_notify`。**加鍵之前先 grep。**
