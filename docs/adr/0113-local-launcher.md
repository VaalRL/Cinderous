# 0113. 本地啟動器（`cinder serve`）：固定 port 的瀏覽器版

- 狀態：已接受（已實作）
- 日期：2026-07-14
- 相關文件：ADR-0098（CLI）、0106（webview 節流實測）、0109（自適應心跳）、
  0111（訊息封存／OPFS）、0112（網頁靜態加密）

## 背景與問題

瀏覽器版其實**早就存在**——桌面 App 有非 Tauri 的分支（`isTauri()` 判斷），行動版本身就是
`react-native-web` 的瀏覽器 App。`vite build` 出來是純靜態 SPA。

缺的只是「怎麼跑起來」。而它有一個**不能隨便處理**的性質：

> **Origin 就是資料的身分。**

`localStorage`、**OPFS 封存**（ADR-0111）都以 origin 為鍵。`localhost:7847` 與 `localhost:3000`
是**兩個不同的 App**。隨手用 `python -m http.server` 或 `vite preview`（每次可能不同 port）
的結果是：**使用者的訊息、聯絡人、封存「憑空消失」**——其實還在，只是在另一個 origin 底下讀不到。

而 ADR-0112 讓這件事更關鍵：資料金鑰由 nsec 導出、密文存在該 origin 的儲存裡。換 origin
＝換一個空白的資料空間。

## 決策

### 1. 固定 port（預設 **7847**）

刻意避開常見的開發用 port（3000／5173／8080／8000），降低撞埠機率——因為撞埠的代價
不是「換個網址」，而是**使用者看不到自己的資料**。

### 2. **撞埠時直接失敗，絕不自動換 port**

這是本 ADR 最重要的一條。

自動換 port 是**最糟**的處理：它會「**成功啟動**」，然後把使用者帶進一個**空白的 App**
——看起來像資料被刪光了。寧可啟動失敗並解釋原因。

撞埠訊息會明講「為什麼不換 port」，而不是丟一句 `EADDRINUSE`。

### 3. `--port` 可覆寫，但 help 與訊息都明講它**會換掉 origin**

有人可能真的必須換 port。那就讓他知道代價。

### 4. 只綁 `127.0.0.1`

不綁 `0.0.0.0`——沒有理由讓區網上的其他機器連進來。

### 5. 路徑穿越防護

`resolveAsset()` 解析後檢查前綴，必須嚴格在 root 底下。擋掉 `..`、**URL 編碼的 `%2e%2e`**、
絕對路徑、Windows 反斜線、NUL 位元組；目錄不算檔案（不列目錄）。

### 6. 不設任何 CORS 標頭

其他 origin 不得讀取本 App 的資源。另加 `X-Content-Type-Options: nosniff`。

### 7. `serve` **不需要私鑰**

它只送靜態資源。**私鑰全程在瀏覽器裡，不經過 CLI。** 故在 `main.ts` 中必須在
`loadNsec()` **之前**分派——否則會平白要求使用者提供 nsec。

## 這台伺服器不持有任何機密

它只送出 App 的靜態資源（**公開的程式碼**）。使用者的資料全在**瀏覽器的儲存**裡，
且自 ADR-0112 起是加密的。這一點值得寫下來：本啟動器**不是**新的攻擊面。

## 誠實告知：瀏覽器版的兩個固有代價（啟動時印出）

### 1. 背景分頁**會被節流**——這是 ADR-0106 的量測結果反過來推

ADR-0106 實測：Tauri 的隱藏視窗**不會**被節流，**因為 WebView2 從未把頁面標記為 hidden**
（`visibilitychange` 從未觸發）。Chromium 的 timer 節流是**以頁面可見性為鍵**的——鍵不成立，
節流不啟動。

但**瀏覽器的背景分頁 `document.hidden = true`** → **鍵成立** → **會被節流**。後果：

- **外送匣泵（200ms，ADR-0041）會停擺** → 送訊延遲
- 心跳被拖（ADR-0109 的自適應已讓它沒那麼嚴重）
- **收訊不受影響**——WebSocket 是網路事件，不受 timer 節流

這是機制上的必然，不是猜測。**桌面版沒有這個問題**（ADR-0106 已實測）。

## 後果

- 正面：
  - `cinder serve` 一行啟動瀏覽器版；不需要 Tauri 工具鏈。
  - 固定 port ＋ 撞埠不換 → **資料不會「憑空消失」**。
  - 測試 +19（路徑穿越：`..`／`%2e%2e`／反斜線／NUL／目錄；SPA 退路；無 CORS；
    405；**撞埠 reject**；參數解析）。

- 負面 / 已知限制：
  - **背景分頁節流**（如上）——這是瀏覽器版的固有代價，無法在應用層繞過。
    若要「關窗仍在線」，用桌面版。
  - 服務的是**桌面 App 的建置產物**（`apps/desktop/dist`）。它在瀏覽器下會走
    `LocalStorage` ＋ OPFS ＋ ADR-0112 的密碼路徑；**沒有 OS 金鑰庫**，所以不設密碼就
    每次要重新輸入 nsec。
  - `--dir` 可指向任何已建置的靜態 App（例如行動版），但預設只指桌面版。
  - 未提供 HTTPS。`http://localhost` **算 secure context**（WebCrypto／OPFS／WebRTC 皆可用），
    故不需要。
