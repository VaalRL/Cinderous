# 0158. 聯絡人標籤補全＋企業頭銜標籤（自填、全組織可見、色彩區隔）

- 狀態：已接受（已實作）
- 日期：2026-07-16
- 相關文件：ADR-0040（本地標籤——本 ADR 補經典佈局入口與清理）、0148（本地暱稱）、
  0154（頭像加密廣播——頭銜複用同一條個人檔管線）、0061（加密個人檔）、0047（名冊）

## 背景與問題

盤點（本次前）：私有標籤只有三欄佈局側欄能貼在聯絡人身上；經典佈局無入口；刪聯絡人
不清標籤（孤兒殘留）；手機版無標籤。另使用者要求新概念——**企業頭銜**：企業模式下
每人**自填**頭銜，全組織看得到，與私有標籤以顏色區隔；私有標籤照常共存。

## 決策

### 1. 經典佈局標籤入口：對話視窗對方頭像旁（ADR-0040 補全）

`ConversationWindow` 的 `.pics` 區（對方頭像下方）新增標籤列：私有標籤 chips（可移除）
＋ 🏷 新增。資料仍是 `nb.groupPrefs`（id 通用，ADR-0040），App 只對 1:1 視窗接線
（群組標籤既有入口不動）。兩種佈局至此都能為聯絡人貼私有標籤。

### 2. 刪除/封鎖聯絡人一併清標籤

App 的統一清理點 `forget()` 追加 `pruneGroup(groupPrefs, pk)`——與離開群組的既有清理
對齊，不再殘留孤兒標籤。

### 3. 手機版＝未來工作（記錄）

行動端標籤顯示/編輯與跨裝置同步（連同標籤鍵按身分命名空間化、隨快照/搬家走）留待
後續 ADR；本次不動。

### 4. 企業頭銜（自填、廣播、色彩區隔）

- **傳輸**：復用 ADR-0154 的加密個人檔管線——`ProfileData` 擴 `title?`（≤24 字，
  收端清洗：收斂空白/截斷；`""`＝移除記號；缺席＝無變更）。中繼站只見密文。
- **可見範圍**：隨個人檔廣播給**該身分的聯絡人**。工作身分的聯絡人由名冊權威管理
  （ADR-0047）＝恰為全組織同事——「企業內大家都看得到」自然成立，無需新協定面。
  企業主的頭銜會廣播給其所有聯絡人（含組織外私人聯絡人），設定處明示。
- **儲存**：`StoredContact.title`（隨聯絡人持久化；體積極小，不需 ADR-0154 的快照剝除）；
  自己的頭銜 `selfTitle` 比照 `selfAvatar` 三態持久化（null 未設／"" 已移除／值），
  隨 `StorageSnapshot` 與搬家捆包走。
- **設定入口**：設定 → 身分 →「頭銜」輸入（僅企業成員與企業主身分顯示；留空套用＝移除）。
  變更比照改名/換頭像清 `profileSentTo` 全量重播。
- **顯示與色彩區隔**：頭銜為**實心強調色 chip**（`chip--role`），私有標籤維持既有
  outline chip——一眼可辨「組織身分」vs「我自己的分類」。顯示位置：三欄側欄列
  （名稱旁）與對話視窗對方頭像旁（標籤列最前）。私有標籤功能不受影響、共存。

## 理由

- 頭銜走個人檔（自填自廣播）而非名冊（管理者簽章）：使用者裁示「每個人可以自填」，
  名冊路徑需經管理者重簽發、違反自助語意；個人檔管線（0061/0154）已有三態語意、
  重播機制與收端防禦，零新協定面（Fix First）。
- 標籤（私有、多值、本機）與頭銜（廣播、單值、隨帳號）是兩個正交概念，分開儲存、
  分開顯示、以色彩區隔——不合併成同一個「標籤」模型。

## 後果

- 正面：兩種佈局都能貼私有聯絡人標籤；刪人不殘留；企業內頭銜互見、與私標一眼區分。
- 已知限制／取捨：
  - 頭銜是自由文字（不做職稱字典/審核）；惡意成員可自稱任何頭銜——名冊的權威名稱
    （管理者定的 `name`）仍是身份錨點，頭銜僅是自述標註。
  - 私有標籤仍為裝置本地全域鍵（`nb.groupPrefs` 不分身分、不同步）——命名空間化與
    同步連同手機版一起留待後續（見決策 3）。
  - 經典佈局群組視窗內無標籤列（群組標籤走清單既有入口）。
- 測試：core（title round-trip/清洗/移除記號）＋engine（儲存三實作、收發廣播、
  快照/搬家攜帶）＋desktop（對話窗標籤列、頭銜 chip 色彩類、設定編輯、清理接線）。
