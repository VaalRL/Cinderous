# 0015. 設定面板：身分備份、桌面通知與未讀（Phase A5）

- 狀態：已接受
- 日期：2026-07-01
- 相關文件：docs/ROADMAP.md（Phase A5）；PRD.md（隱私預設）；docs/adr/0014（聯絡人管理）

## 背景與問題

Phase A5 要補齊「設定與狀態」的產品化缺口：使用者需要能備份身分（否則清除瀏覽器資料即永久失去帳號）、在視窗未聚焦時收到提醒、並看到哪些對話有新訊息。這些都涉及隱私與私鑰處理，需明確界定。

## 決策

- **身分備份（私鑰顯示）**：於設定面板提供「顯示私鑰（nsec）」，預設隱藏，點擊才顯示，並附強烈警語（等同帳號、切勿外流）。私鑰**僅在本機**於使用者主動操作下呈現與複製，**不經任何網路傳送**，符合 SSOT 硬規則「明文與私鑰不離開裝置」。後端以唯讀 `selfNsec` 提供（示範模式不提供）。
- **桌面通知**：採用瀏覽器 `Notification` API，預設關閉；啟用時才請求權限，偏好存 `localStorage`（`nb.notify`）。僅在 `document.hidden`（視窗未聚焦）且收到他人訊息時觸發，通知內容為訊息文字（本機呈現，不外送）。
- **未讀徽章**：以每位聯絡人的未讀計數呈現；僅在 `document.hidden` 時累加，視窗重新聚焦（`visibilitychange`）即全部清除，開啟該對話亦清除其計數。定義單純、可預期。
- **音樂狀態輸入**：於個人區提供輸入口，`blur`／Enter 時呼叫既有 `setNowPlaying`（Ephemeral，不持久），與好友端的 `nowPlaying` 顯示對應。

## 後果

- 正面：使用者可安全備份身分且流程明確；通知與未讀提升可用性且不犧牲隱私（皆本機處理）。設定面板以能力偵測顯示（示範模式隱藏私鑰備份）。
- 負面 / 未來：私鑰以明文顯示於畫面存在肩窺（shoulder-surfing）風險，故預設隱藏並警示；Tauri 版應改由 OS 金鑰庫（ADR 待補）託管、以更安全方式匯出。**連線/重連中狀態**未納入本次（需連接器層重連 + 狀態回報），列為 A5 後續。多設備未讀/通知一致性另議。
