# 0143. 三欄中欄的內嵌對話：整塊填滿、去除浮動視窗外觀

- 狀態：已接受（已實作）
- 日期：2026-07-15
- 相關文件：ADR-0079（桌面佈局 modern 三欄）、0104（原生拖放命中測試），
  apps/desktop/src/ui/ConversationWindow.tsx、msn.css

## 背景與問題

三欄（modern）版中欄應該**整塊都是對話**，但實際上對話仍以「傳統對話框」呈現——一個帶
**外框、視窗底色（`--chrome`）、彩色視窗標題列（`--titlebar` 漸層）**的浮動視窗，飄在中欄裡。

`.convo--embed` 當初只去掉了圓角與陰影，**沒去掉外框、視窗底色，也沒改標題列**，所以還是一副
視窗樣。加上中欄上方已有 `DeckTabs`（分頁列顯示對話名），等於「分頁 ＋ 又一層視窗標題列」的雙重
標頭，更像視窗中的視窗。

另有一個獨立 bug：`ConversationWindow` 的 `.convo-dock` 裡有一行 `// …` 註解**寫在 JSX 子節點位置**
——JSX 會把它當**文字**渲染出來，於是每個對話上緣多一串「// data-convo…」的雜訊文字。

## 決策

1. **內嵌對話整塊填滿、去視窗外觀**：`.convo--embed` 加上 `border: none; background: var(--panel)`
   （原本已 `border-radius:0; box-shadow:none`）——不再有外框/圓角/陰影/視窗底色。
2. **標題列改為扁平對話標頭**：`.convo--embed .win__title` 由彩色視窗標題列改為 `--surface2` 底、
   `--ink` 字、底部 1px 分隔線、無文字陰影；標題列按鈕改 `--muted`/hover `--ink`。它成為對話的
   工具列標頭（含通話/成員/離開等鈕），而非視窗標題列。
3. **修正 JSX 註解 bug**：`// …` 改為 `{/* … */}`，不再把註解當文字渲染。

經典（floating windows）版不受影響——那裡對話本來就該是浮動視窗（`.convo` 無 `--embed`）。

## 後果

- 正面：三欄中欄的對話**整塊填滿、扁平標頭**，不再像浮動視窗；上緣雜訊文字消失。
  ConversationWindow 測試 18 全綠（純 CSS ＋ 註解修正，無行為改動）。
- 已知限制：中欄仍是「DeckTabs 分頁列 ＋ 對話標頭」兩層；標頭保留是因為它承載通話/成員/離開等動作。
  若日後要把標頭動作併入分頁列、再省一層，另議。
