# 0078. 副色（次要主題色）：標題列 ＋ 頂部漸層

- 狀態：已接受
- 日期：2026-07-10
- 相關文件：docs/adr/0064（自訂主色——本 ADR 在其上擴充）、apps/desktop/src/accent.tsx、
  apps/desktop/src/ui/msn.css、apps/desktop/src/ui/SettingsPanel.tsx

## 背景與問題

ADR-0064 提供**單一 `--accent`（主色）**自訂，整個 UI 由其 `color-mix` 衍生。使用者要
「主色＋副色」**雙色**識別；經確認，副色驅動**視窗標題列與頂部背景漸層**，其餘（送出泡泡、
主按鈕、連結、選取）仍由主色。

## 決策

1. 新增 **`--accent2`（次要主題色）**，CSS 預設 `--accent2: var(--accent)`——**未設＝沿用主色**，
   完全向後相容。
2. **重新導向**：`--titlebar` 與頂部背景漸層 `--bg-a/b/c` 改由 `--accent2` 以 `color-mix` 推導。
   （頂部漸層本為固定藍、現改為隨色系連動，順帶修正「自訂主色時背景不跟著變」的小不一致。）
   其餘 accent 面（泡泡、按鈕、連結、hover-on-accent）仍吃 `--accent`。
3. `accent.tsx` 加 `accent2` 狀態（localStorage `nb.accent2`）：設值時 root 套 `--accent2`
   （深色同樣經 `accentForTheme` 提亮以維持對比）；未設則移除該屬性 → 回退 `var(--accent)`。
4. 設定面板「主題色」區加第二排「副色」取色器：共用預設色票 ＋ 自訂取色 ＋「跟隨主色」重設。
   純本地儲存、不上雲。

## 理由

- 延續 ADR-0064 的 token 化與本地儲存取向（Fix First）；副色可留空 → 向後相容且零風險。
- 讓「外框（標題列/背景）」與「內容 accent」可分離為兩色，達成雙色識別，同時把原本固定的
  背景漸層納入色系連動。

## 後果

- 正面：雙色個人化；頂部漸層終於隨色系連動、與標題列一致。
- 負面／已知殘餘：預設主色下，背景漸層由「固定藍」改為「主色衍生」（近似色、視覺變化極小）；
  多一組本地設定（`nb.accent2`）。副色與主色一樣**不隨換機同步**（本地限定，比照 ADR-0064）。
