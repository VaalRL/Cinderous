# 0115. 行動端功能對齊（二）：建立群組與成員管理、敲一下、上線狀態

- 狀態：已接受（已實作）
- 日期：2026-07-14
- 相關文件：ADR-0027（群組：無共用金鑰的成對扇出）、0088（在線元數據最小化／隱身）、
  0095（群組回條分級）、0114（行動端對齊一）

## 背景與問題

ADR-0114 修好了「點進群組送訊會爆」，但行動端仍**只能被邀請入群，不能開群**
——「能用群組但不能建群組」很跛。另外兩個小缺口：**敲一下只能收不能發**、
**沒有上線狀態**（只有隱身）。

引擎的 API 全都在（`createGroup` / `leaveGroup` / `removeGroupMember` / `sendNudge` /
`setStatus`），且 `onGroups` 會回放——純 UI 接線。

## 決策

### 1. 建立群組：聊天清單的 `#` 入口

名稱（可留空，引擎會給預設名）＋ 從聯絡人挑成員（chip 多選）。

**至少要有一位成員才能建立**——空群沒有收件人，群訊會被 `trackFanout()` 直接標成
`failed`（ADR-0095：`events.length === 0` ⇒ 送不出去）。故：**沒有聯絡人就不顯示建群入口**，
且成員為空時送出鈕停用。已有測試釘住。

（這也是為什麼「成員清單」就是「收件人清單」——群組**無共用金鑰**（ADR-0027），
對每位其他成員各包一個 Gift Wrap 扇出。）

### 2. 群組管理：對話頂列的 👥

- **任何成員**都能離開群組（離開後自動退回主畫面——已經不是成員了，留在對話畫面沒有意義）。
- **只有管理者**能移除成員（ADR-0027）。UI 依 `group.admin === selfPubkey` 顯示；
  **真正的強制點在引擎**（`removeGroupMember` 檢查 `group.admin !== this.self.pubkey` 就 return）
  ——UI 只是不顯示，不是安全邊界。
- 成員面板**要點開才出現**（不佔掉對話空間）。

### 3. 敲一下：1:1 才有

敲一下是**定址給單一收件人**的 ephemeral 事件（`#p`），群組沒有這個語意 → 群組不顯示。
過去行動端只接了 `onNudge`（收），沒有發送入口。

### 4. 上線狀態：online / away / busy

與桌面同一組。**隱身（ADR-0088）優先**——隱身時後端完全不廣播心跳，狀態設定不生效。

## 後果

- 正面：
  - 行動端可**建立群組、管理成員、離開群組**。
  - 敲一下可發；上線狀態可設。
  - 測試 +9（建群入口的三種情境、1:1 不顯示群組選單、成員面板需點開、
    敲一下在群組不顯示…）。全 repo **919 通過**。

- 行動端**仍缺**（記錄在案）：
  - 通知（Web Notifications）
  - 多身分、本地密碼
  - 貼圖、相簿、AI
  - 音樂狀態
  - 多中繼管理、配對搬家（送出端）
  - **公告群**（ADR-0049 `announce`）與組織名冊（ADR-0047）——行動端不建這類群，
    但**能接收**（引擎照常處理）。

- 一個 i18n 教訓：`group_name` / `group_members` / `group_create` **早就存在**（桌面在用）。
