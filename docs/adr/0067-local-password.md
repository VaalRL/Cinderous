# 0067. 本地密碼：以密碼衍生金鑰包裹身分祕密（否決 nsec 日常登入）

- 狀態：已接受
- 日期：2026-07-09
- 相關文件：docs/adr/0015（身分備份 nsec）、0045（多身分設定檔）、0053（OS 金鑰庫）、
  0054（加密儲存基質 AES-256-GCM）、0055（禁止跨身分互加／分身不可連結）、0066（relay 搬家）、
  apps/desktop/src-tauri/src/keyvault.rs、apps/desktop/src/storage/profiles.ts

## 背景與問題

同一台設備多人共用（家庭電腦、共用工作機）時，Cinder 目前的信任邊界是**作業系統帳號**：

1. `switchProfile` 切換身分**無任何驗證**——點一下就進入別人的身分、看到全部歷史。
2. OS 金鑰庫（Windows Credential Manager，ADR-0053）在**同一個 Windows 工作階段內
   可被靜默讀出**：任何在此帳號下執行的程式都能列舉並取得 nsec 與 `db:` 資料金鑰
   （本專案即以 `cmdkey /list` 實際驗證過）。
3. 資料檔（`<namespace>.enc`，ADR-0054）的解密金鑰就放在金鑰庫——拿到金鑰庫等於拿到全部。

亦即：只要共用 OS 帳號，「別人的身分」形同不設防。需要一道**身分層級**的防線。

同場檢討一個替代提案：**「直接用 nsec 當日常登入憑證」**（以私鑰取代名稱＋密碼）。
技術上可行——nsec 可還原身分、天然全域唯一——但作為**日常**憑證有結構性缺陷，見選項 C。

## 考量的選項

- 選項 A：**純 UI 鎖**（密碼只擋畫面，資料與金鑰不動）——否決。金鑰庫與 `.enc`
  仍可被同帳號程式直接讀取，是假安全感；違反「可測、誠實」原則。
- 選項 B：**密碼衍生金鑰包裹身分祕密**（採用）——密碼經 KDF 衍生 KEK，
  包裹該身分的 nsec 與資料金鑰；不輸入密碼就**數學上**解不開。
- 選項 C：**nsec 直接作日常登入**——否決：
  - 日常高頻輸入迫使使用者把 nsec 放在剪貼簿／記事本，Windows `Win+V`
    剪貼簿歷史會留存；等於訓練使用者養成「到處貼主金鑰」的習慣。
  - 被側錄的後果不對稱：密碼外洩＝單機單身分受害、可換；nsec 外洩＝
    身分本體永久失竊、**不可輪替**（換 nsec＝換身分，聯絡人全斷）。
  - 與 ADR-0015 的備份警語（「絕不要輸入到任何地方」）自相矛盾。
- 選項 D：**要求每人各開 OS 帳號**——不現實：家庭共用機常態就是單一帳號；
  把安全性外包給使用者做不到的前提，等於沒有防線。

## 決策

採選項 B，**每個身分可獨立選配**的本地密碼（預設關閉，不加負擔於單人設備）：

1. **金鑰包裹**：啟用時以 Argon2id（記憶體硬 KDF，抗 GPU 暴力）自密碼衍生 KEK，
   以 AES-256-GCM 包裹該身分的兩把祕密——nsec 與 `db:<namespace>` 資料金鑰——
   包裹後的密文取代金鑰庫中的明文條目；鹽與 KDF 參數隨密文存放。
   解鎖＝輸入密碼 → 衍生 KEK → 解開兩把金鑰進記憶體，落地永遠是密文。
2. **隱藏身分選項**：啟用密碼的身分可加選「不在切換器顯示」——共用設備上
   他人連「這個身分存在」都看不到，與 ADR-0055 的分身不可連結一致。
3. **閒置自動上鎖**：重用既有閒置偵測（ADR-0030 的 idle 機制），逾時清除
   記憶體中的金鑰並回到解鎖畫面。
4. **啟用時強制 nsec 備份**：啟用密碼前必須完成一次 nsec 備份確認（沿用
   ADR-0015 的備份流程）——因為忘記密碼後**本機無任何後門**（見後果）。
5. **改密碼＝重包裹**：僅重新包裹兩把金鑰，資料檔不需重加密（資料金鑰不變）。

**nsec 的角色定位**（與 ADR-0066 一致）：nsec 是**主金鑰**，僅用於
「首次匯入／換機／搬家／忘記密碼救援」，不作日常登入憑證；日常解鎖用本地密碼。

## 理由

- 只有「密碼參與加密」才是真防線：選項 A 擋得住手滑、擋不住任何程式碼；
  選項 B 讓沒有密碼的攻擊者拿到金鑰庫與 `.enc` 也解不開。
- Argon2id 是目前密碼雜湊的共識選型（OWASP 首推），記憶體硬特性正對
  「離線暴力破解包裹密文」這一威脅。
- 每身分獨立選配維持「本地優先、零強迫」：單人設備不受影響，共用設備各自加鎖。
- 否決 nsec 日常登入把「主金鑰」與「日常憑證」分層——與銀行「印鑑 vs 提款卡密碼」
  同構：可輪替的低價值憑證日常用，不可輪替的主金鑰鎖進抽屜。

## 後果

- 正面：共用設備上身分互相隔離；金鑰庫被讀走也無法解密；密碼可輪替、
  外洩損害有限；隱藏身分讓「存在性」也受保護。
- 負面／已知殘餘風險：
  - **忘記密碼＝本機永久不可解**，僅能憑 nsec 備份重建身分（歷史若無多設備
    冗餘即遺失）——文案必須誠實說明，並以啟用時強制備份緩解。
  - 不防執行中的惡意軟體（解鎖期間金鑰在記憶體、鍵盤側錄可截密碼）——
    威脅模型文案不得誇大為「防毒級」保護。
  - 新增 Rust 依賴：`argon2` crate（KDF 在原生層執行，避免 JS 端計時/記憶體弱點）。
- 後續行動 / 待辦：依 ROADMAP Phase H 的 H4 施工；完成後更新 ARCHITECTURE.md
  金鑰管理一節與 `docs/SECURITY.md` 威脅模型。
