# 0215. 經典佈局：自訂分組排序切換 ＋ 可收合區塊標頭（MSN 擬真）

- 狀態：已接受
- 日期：2026-07-20
- 相關文件：ADR-0079（三欄佈局）、ADR-0040/0158（標籤/頭銜）、ADR-0214（統一列＋狀態訊息上列）、`apps/desktop/src/ui/contact-grouping.ts`、`ContactListWindow.tsx`

## 背景與問題

對照真正的 MSN Messenger，經典佈局（`ContactListWindow`）最大的兩項落差：
- **A**：聯絡人**只能依上線狀態分區**，無「自訂分組」，也不能切換排序（MSN 可依 群組/狀態/名稱）。
- **B**：區塊標頭**不可收合**，計數是「該狀態人數」而非 MSN 招牌的「**線上/總數 (2/5)**」。

## 考量的選項

分組歸屬模型（使用者選定）：
- **沿用既有標籤·可多屬（採用）**：直接把 ADR-0040/0214 的**標籤**當分組維度；一人多標籤 → 多分組區都出現；無標籤 → 未分組。零新資料模型、零遷移。
- 沿用標籤但每人只顯示第一個標籤（不重複，但失去 MSN 多群歸屬）。
- 新增單一 `contact.group` 欄位（每人一組，最貼 MSN 預設，但要新資料模型＋遷移，工程量最大）。

## 決策

採用「沿用標籤·可多屬」，並把分組/排序邏輯抽為純函式：

- **`contact-grouping.ts` `groupContacts(contacts, mode, contactLabels)`**（純函式、可測）→ `sections[]`：
  - `status`：線上→離開→忙碌→離線、區內名稱排序、標頭顯示總數（沿用既有分區）。
  - `group`：每標籤一區（可多屬）＋未分組 catch-all；區內**線上優先**再名稱；標頭顯示「**線上/總數**」。
  - `name`：單一「全部」區、A→Z、標頭顯示總數。
  - 線上定義＝`status !== "offline"`（MSN 語意：away/busy 仍算已連線）。
- **排序切換控制**（依狀態/分組/名稱），預設「依狀態」（不驚動現有使用者）；選擇持久化 localStorage。
- **可收合區塊**：每區標頭加 ▸/▾（`SectionHead`），收合狀態持久化 localStorage（per-section key）；套用到 狀態/分組/名稱區、群聊區、封鎖區（**不含訊息請求區**——待裁示動作不該被收起藏住）。
- **範圍**：只做經典版；三欄版維持最近排序＋搜尋（ADR-0079 雙模式）。

## 理由

- **Fix First / 零新資料**：MSN 的「分組」＝把聯絡人歸類，Cinder 的標籤已能做到（且 ADR-0214 已把 🏷 入口接進經典聯絡人列）。A 於是變成「用既有標籤分區顯示」，不引入平行的「分組」概念。
- **可多屬更貼 MSN**：MSN 允許一人屬多群；沿用標籤天然支援。
- **純函式可測**：`groupContacts` 與計數邏輯與 UI 解耦、易測易維護。
- **不驚動現有使用者**：預設仍「依狀態」，收合/排序皆為漸進式增強。

## 後果

- 正面：經典佈局可依 群組/狀態/名稱 切換、標頭可收合、分組顯示「線上/總數」——一眼更「像 MSN」；分組直接復用標籤，無新資料。
- 負面 / 已知殘餘風險：「依分組」需先用 🏷 為聯絡人上標籤，否則全落「未分組」（可接受，MSN 也需自建群組）；空標籤群不顯示（v1）。三欄版不套用此切換（刻意）。
- 後續行動 / 待辦：可選讓群聊列/封鎖列的收合鍵與 UI 一致化；可選加「新增分組」捷徑（本質＝新增標籤）；未分組是否可隱藏由使用者決定。
