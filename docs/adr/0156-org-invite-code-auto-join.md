# 0156. 入職邀請碼與自動入職——一鍵複製、貼上即加入、名冊自動核准

- 狀態：已接受（已實作）
- 日期：2026-07-16
- 相關文件：ADR-0155（企業主身分——邀請碼由其發出）、0047（組織名冊——本 ADR 自動化其
  佈建迴路）、0070（備份碼——單一字串可貼/可印的先例）、0121（訊息請求——陌生人邊界）

## 背景與問題

ADR-0155 之後，建立組織的流程仍有兩次**帶外 npub 交換**：企業主把「relay 網址＋自己的
npub」發給員工、員工把自己的 npub 回傳、企業主手動貼進名冊重發。使用者裁示：企業主
**一鍵複製**一段組合文字；員工**建立企業成員身分時貼上**、或**在登入畫面直接貼進顯示
名稱欄**即自動解析、自動建立成員身分；員工身分資料**自動回傳**企業主、**自動完成加入**。

## 決策

### 1. 入職邀請碼（core `invite.ts`）——單一 token、可嵌在任意文字中

`cinderinvite1<base64url(JSON{v:1, relayUrl, adminPubkey, token})>`：

- **不是機密**（relay 與 npub 本來就要交付），但 `token` 是**核准權杖**（capability）：
  沒有它的入職請求一律被忽略——撿到管理者 npub 的人不能憑空入冊。
- `parseOrgInvite()` 以正則在**整段貼上的文字中**抽出 token（員工貼整封邀請信也能解析）；
  欄位驗證（wss:// 前綴、64 hex pubkey）不符回 null。
- 權杖於**建立企業主身分時**生成（`newInviteToken()`，128-bit 隨機 hex），存
  `Profile.orgInviteToken`。同一組織固定一碼；洩漏時的「重設邀請碼」為後續（新 ADR）。

### 2. 入職請求（core rumor kind `ORG_JOIN = 42`）——加密、可離線

`wrapOrgJoin({name, token}, sk, adminPk)`：Gift Wrap 給管理者（7 天過期、走離線信箱）。
中繼站只見密文；管理者離線也收得到，下次開機補核准。

### 3. 成員端：貼碼即自動化（三個入口）

- **登入畫面**：顯示名稱欄偵測到邀請碼 → 轉為「加入組織」面板（填自己的顯示名稱＋
  瀏覽器本地密碼）→ 建立 `enterprise` 成員身分（relay/管理者自動帶入）並記
  `Profile.orgJoinToken`。
- **新增身分 → 🏢 企業成員**：表單頂端貼碼欄，貼上即自動填 relay＋管理者並帶權杖。
- **開機重送直到收錄**：成員後端每次啟動時，若名冊尚未包含自己 → 送一次入職請求
  （冪等：管理者對已在冊者忽略）。管理者離線、先建成員後建名冊等時序全部自癒。

### 4. 企業主端：自動核准

- 企業主後端（`orgOwner` 旗標）**訂閱自己簽章的名冊**（kind 10038 可取代事件）——重啟後
  從中繼站找回 `lastRoster`，自動核准不因重啟失憶。
- 收到入職請求：權杖不符 → 忽略；已在冊 → 忽略；否則**自動併入名冊重新簽發**，並
  **自動把新成員加為聯絡人**（互送加密個人檔——名字/頭像立即互通）。
- 首份名冊尚未發佈時先排隊（`pendingJoins`），發佈時自動併入。
- 名冊一重發，全體成員經 ADR-0047 對帳自動長出新同事。

## 完整流程（實作後）

```
企業主：＋ → 🗂 企業主 → 名冊管理「複製邀請碼」→ 貼給員工（信件/IM/紙本皆可）
員工　：登入畫面貼上邀請碼 → 填顯示名稱 → 完成
（其餘全自動：入職請求 → 權杖驗證 → 名冊併入重發 → 全公司通訊錄同步）
```

## 理由

- 權杖讓「全自動核准」不變成「撿到 npub 就能入冊」；密文入職請求維持隱私鐵則。
- 可取代名冊事件既有機制（ADR-0047）承擔了「重啟找回狀態」——零新增持久化面。
- 開機重送＋在冊即忽略的冪等設計，把所有時序競態（管理者離線、先員工後名冊）化為自癒。

## 後果

- 正面：建組織從「兩次帶外交換＋手動貼名冊」變成「一鍵複製＋貼上即加入」。
- 已知限制／取捨：
  - 邀請碼＝長期有效的核准權杖，存於 `nb.profiles`（明文登錄）；洩漏即任何人可入冊，
    目前的補救是換一個企業主身分。「重設/失效邀請碼」與「入職待核清單（手動模式）」留待
    後續 ADR。
  - 名冊自動重發以 `updatedAt = now` 簽發；多裝置同時核准以較新者收斂（NIP-01 取代語意）。
  - 行動端未接（登入/新增身分皆桌面先行；同 ADR-0145/0155 的既有限制）。
- 測試：core（邀請碼 round-trip/文字抽取/壞輸入；入職請求 wrap/parse）＋engine（貼碼建立
  →自動核准→雙向通訊錄整合流）＋desktop（登入面板/表單貼碼/複製鈕 SSR）。
