> 本文件是一份**研究記錄**，結論為「暫緩」。目標是盤點本產品可能的營利面、釐清授權能與
> 不能達成什麼，並提出防止「他人直接營利」的**非授權**手段（商標）。日後若決定推進商標

## 背景與問題

專案立場：**本專案不進行營利**，僅允許使用者**直接對節點營運商捐獻**（PRD §12、ADR-0089/0090）。
需求：(a) 確保**作者/官方自身不營利**；(b) 確保**他人無法直接拿本產品營利**。本 ADR 檢查
現況能否達成這兩點。

## 關鍵事實：AGPL 不禁止營利

授權為 **AGPL-3.0-only**。AGPL 前言明載「free 指自由、非價格……你有自由散布拷貝**並收費**」。
AGPL 的槓桿是**兩件事，皆非禁止營利**：

1. **Copyleft（傳染性）**：衍生版須維持 AGPL 開源——他人**不能閉源圈地**成專有產品。
2. **網路服務源碼揭露**：拿去架 SaaS 者須對其使用者公開修改後源碼。

⇒ AGPL 擋得住**專有化圈地**，擋不住**收費本身**。這是「禁止他人營利」與「維持 AGPL 精神」
之間的根本矛盾。

## 目標 (a)：作者/官方不營利——已結構性達成

- **零站內金流**：無站內錢包/托管/抽成；捐款一律**外部連結導流**（ADR-0089，桌面角落卡、
  可關閉、行動端刻意不放）。
- **官方捐款可審計**：官方款項走**簽章式資金透明度**（ADR-0090，`verifyFunds` 對釘死公鑰
  驗簽、算 runway、fail-closed）。錢碰官方的唯一路徑被透明機制盯著。

此塊無需再補。

## 目標 (b)：他人直接營利——盤點（AGPL 皆不禁止）

| 營利方式 | 可行性 | 現有防線／缺口 |
|---|---|---|
| 架付費 relay／托管 SaaS | 高（最現實） | NIP-42 AUTH 可做付費帳號；企業套件（allowlist、`MAX_TTL_DAYS`、`MAX_FILE_MB`、名冊/託管/儲存槽）近乎可賣 B2B。AGPL 只要求公開源碼，不阻止收費 |
| 改名重包裝上架收費 | 高 | 僅**商標**能擋；改名 fork 販售 AGPL 允許（須附源碼） |
| fork 加站內營利（廣告／付費貼圖商店／付費認證／付費頭像狀態） | 高 | PRD 只約束**官方產品**；fork 可任意加。貼圖系統、頭像/狀態廣播、認證概念皆現成加裝點 |
| 賣元資料／分析 | 中 | Gift Wrap 藏訊息社交圖譜，但**上線訂閱洩漏聯絡人集合、IP、時序**（見 observability 文件）；營利型 relay 可變現 |
| 賣支援／部署／客製 | 高 | 開源標準商業模式，擋不了也不該擋 |
| 假冒官方收捐款 | 低 | 被**商標**＋ADR-0090 簽章透明度擋住（官方款項可驗簽，冒充可偵測） |

## 建議：能加的防線都不是授權能做的

1. **註冊商標「Cinder」與 Logo**：**唯一**能擋「改名/冒充販售」的合法工具。商標與著作權授權
   分離——原始碼 AGPL，他人仍不得用你的名稱/標誌販售或暗示官方背書。**投報率最高**。
2. **商標政策文件**：明訂「fork 必須改名、不得暗示官方背書」，並定義允許的名義使用範圍。
3. **接受 AGPL 現實**：copyleft 已是最強實務嚇阻——商業 fork 必須公開源碼、無法建閉源護城河，
   商業誘因大幅削弱。

## 若「任何人都不得營利」是硬需求（備選，代價大）

改用**非商業 source-available 授權**（PolyForm Noncommercial／BSL／程式碼 CC-BY-NC）。代價：

- **不再是 OSI/FSF 定義的「開源」**，牴觸 PRD 反覆強調的「去中心化與 AGPL 精神」；
- 影響社群自架 relay 的志願生態（許多人不碰非商業授權）；
- 執法困難。

## 決策（暫緩）

- **維持 AGPL-3.0 為主**；「自己不營利」已結構性達成，copyleft 阻止閉源圈地。
- **待辦（暫緩，非本次實作）**：評估**註冊 Cinder 商標＋Logo**、撰寫**商標政策**——這是唯一
  能有效擋「冒充官方／改名販售」的手段。
- **不採**非商業授權（除非產品定位改變）：其代價是離開開源，與專案核心精神相斥。
- **不新增**任何站內金流/營利面（維持 PRD §12 排除清單）。

## 後果

- 正面：釐清授權的能與不能，避免誤以為 AGPL 能「禁止營利」；把可行防線收斂到商標。
- 已知限制：商標註冊需法律/行政資源與管轄權考量（跨國執法有限）；即使註冊，仍無法阻止
  「合法改名的商業 fork」——那是開源的固有代價，非缺陷。
