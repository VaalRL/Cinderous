# 0132. 圖片分享——行動端走原生分享選單、桌面走快速複製（圖片／路徑）

- 狀態：已接受（已實作）
- 日期：2026-07-15
- 相關文件：**ADR-0023（媒體相簿）**、0093（收檔另存）、0102（縮圖＋讀回原檔）、0128（讀檔白名單）

## 背景與問題

收發的圖片能看（相簿／燈箱／縮圖，ADR-0023/0102），但**沒有任何「分享」出口**——不能把一張
收到的圖片轉出去、複製、或存到系統。使用者要圖片，得自己在對話裡重新傳一次。

兩個平台的自然體驗不同，硬套同一套會兩邊都彆扭：

- **手機**：其他 app 的體驗是**原生分享選單**（一鍵叫出「傳到 LINE／存到相簿／…」）。
- **桌面**：沒有系統分享選單的文化，自然的是**快速複製**——複製圖片本身、或複製它的檔案路徑。

而且**行動端根本還沒有相簿**（ADR-0023 只做了桌面）。所以「分享」不能綁在相簿上，否則行動端
就無從分享。

## 決策

### 1. 分享掛在**圖片訊息**上，不綁相簿

分享的對象是「一張圖片」，而圖片訊息兩端都已渲染（行動端 `<Image>`、桌面內嵌縮圖＋燈箱）。
掛在訊息上 → **行動端不必先蓋相簿**也能分享，桌面則在既有燈箱加動作。零新協定、零後端狀態，
站在 ADR-0093（傳檔）／0102（縮圖、讀回原檔）之上。

### 2. 行動端：原生分享選單（Web Share API）

圖片訊息下方一個「分享」按鈕 → `navigator.share({ files: [File] })`。在手機瀏覽器／webview 上，
這會叫出**作業系統的分享選單**——與其他 app 同一套體驗。位元組取自已渲染的圖片
（`f.url` 本 session 原圖，否則 `f.thumb` 跨 session 縮圖），`fetch` 成 Blob → `File`。

- 先 `navigator.canShare({ files })` 探測；不支援（部分桌面瀏覽器）→ 退回既有的下載
  （`saveFile`），不讓按鈕變成死路。

### 3. 桌面：快速複製（圖片／路徑）

燈箱（點圖放大的檢視）加兩個動作：

- **複製圖片**：把整張圖寫進剪貼簿。為相容性一律**轉 PNG**（Chromium/WebView2 的剪貼簿只收
  `image/png`）——載入 `src` 到 canvas → `toBlob("image/png")` → `ClipboardItem`。
- **複製路徑**：把 `savedPath` 寫進剪貼簿（`writeText`）。**僅在已另存**（有 savedPath）時顯示
  ——沒存過的圖沒有路徑可複製。

複製後給一個短暫的「已複製」回饋。

### 4. 平台能力放 `native/`，UI 只呼叫

- 桌面 `native/clipboard.ts`：`copyImageFromUrl(src)`、`copyText(text)`。
- 行動端 `native/share.ts`：`shareImageFromUrl(src, name, mime)`（回 `boolean`：是否成功走了原生分享）。

UI 不直接碰 `navigator.*`——平台差異收在 `native/`，元件只呼叫（比照 `save-file.ts`／`files.ts`）。

## 理由

- **分享的對象是圖片、不是相簿**。把它綁在訊息上，行動端立刻能分享（不必等相簿），桌面則自然
  落在燈箱。這也讓「行動端相簿」成為**獨立**的後續，而非分享的前置。
- **兩端用各自的原生慣例**：手機的分享選單、桌面的複製。硬套同一套只會兩邊都不像原生。
- 位元組來源沿用 ADR-0102 的兩層（原圖／縮圖），不新增儲存或協定。

## 後果

- 正面：
  - 行動端圖片可**一鍵原生分享**（傳給別的 app／存系統相簿由 OS 分享選單接手）。
  - 桌面圖片可**快速複製圖片本身或其路徑**。
  - 純前端／原生能力，零協定、零後端、零新儲存欄位。
  - 測試 +18（桌面 native `clipboard` 6：複製文字有/空/不支援、複製圖片轉 PNG/canvas 失敗/無
    ClipboardItem；行動 native `share` 6：canShare 分流/退回下載/取消不外洩例外；桌面燈箱 UI 3：
    複製鈕恆在、路徑鈕僅在有 savedPath、點擊呼叫剪貼簿層；行動 UI 3：圖片訊息即使無回應/收回仍可
    長按分享、純文字/已收回不可）。

- 已知限制：
  - **行動端跨 session 只分享得到縮圖**（原圖 blob 是本 session 的；行動端不像桌面有 savedPath
    可讀回原檔）。這是 ADR-0102 在行動端的既有限制的延伸，非本 ADR 引入。
  - **桌面「複製圖片」在 WKWebView（macOS）上支援較弱**——WebView2（Windows）與 Chromium
    瀏覽器可靠。best-effort ＋ 失敗回饋，不改用額外的 Tauri 剪貼簿外掛（避免相依膨脹）。
  - 只做**圖片**分享（對齊本次需求）；機制可延伸到任意檔案，另議。
  - 「存到系統相簿」等**真正原生**的動作，在 web 環境由分享選單代勞；EAS 原生建置可再接
    expo Sharing／MediaLibrary（`files.ts` 已預留註記）。
