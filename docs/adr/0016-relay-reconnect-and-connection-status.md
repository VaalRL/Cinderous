# 0016. 中繼站自動重連與連線狀態（Phase A5）

- 狀態：已接受
- 日期：2026-07-01
- 相關文件：docs/ROADMAP.md（Phase A5）；docs/adr/0015（設定與狀態）；ARCHITECTURE.md §6（連線/心跳）

## 背景與問題

前端 `webSocketConnector` 原本只開一次 WebSocket，斷線後不重連，UI 也無法得知連線是否正常。要讓桌面前端「真的能用」，需在網路波動或中繼重啟後自動恢復，並讓使用者看到「連線中／已斷線重連中」。

## 決策

- **自動重連**：`webSocketConnector` 於 `close` 事件以**指數退避**重連（`1s、2s、4s…` 上限 15s；成功後重置），`error` 轉為 `close` 觸發重連。斷線期間送出的訊息暫存於 `pending`，連上後一次補送。
- **狀態回報**：連接器新增可選 `onStatus(state)` 回呼（`connecting`／`online`／`offline`）。`RelayConnector` 型別擴充第二參數，測試替身（in-memory）可忽略。
- **重連後重訂閱**：`RelayClient` 不會自動重送訂閱，故後端在收到 `online` 時重新 `resubscribe()` 並發送心跳，確保重連後仍收得到好友心跳/私訊。
- **UI**：連線狀態非 `online` 時，聯絡人視窗頂端顯示提示條（連線中／已斷線重連中）；示範（瀏覽器）模式恆為 `online`，不顯示。

## 後果

- 正面：中繼重啟或網路抖動後可自動恢復，符合「本地優先、低延遲」與「可用性」目標；狀態對使用者透明。連接器層純以事件驅動，狀態經真實 relay E2E 驗證（好中繼無提示條、壞位址顯示重連中）。
- 負面 / 未來：目前重連無「使用者主動停止」的關閉路徑（元件卸載後背景仍會嘗試重連，與改版前同屬既有行為），Tauri 版將由 Rust 背景長連線（ADR/M1）接管、含更完整的生命週期與睡眠喚醒策略。退避參數未來可依 `docs/adr/0006` 容量校準調整。
