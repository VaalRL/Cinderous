# 0073. 忘記密碼救援：nsec 解鎖並重設密碼（修訂 ADR-0067 後果）

- 狀態：已接受
- 日期：2026-07-10
- 相關文件：docs/adr/0053（金鑰庫）、0054（加密儲存基質）、0067（本地密碼——本 ADR
  修訂其「本機永久不可解」後果條款）、0070（加密備份碼）、
  apps/desktop/src-tauri/src/passlock.rs、apps/desktop/src-tauri/src/main.rs、
  apps/desktop/src/ui/UnlockScreen.tsx

## 背景與問題

ADR-0067 的本地密碼把「資料金鑰」只用密碼衍生金鑰（Argon2id KEK）包裹，因此
**忘記密碼＝那台機器的本地資料（`.enc`）永久不可解**——即使使用者手上握有 nsec。
這比必要更嚴苛：ADR-0067 已**強制**啟用密碼前備份 nsec，而 nsec 是主金鑰
（能拿到它的人本就能讀你所有訊息、拉雲端快照）。既然如此，讓 nsec 也能救回本機
資料，並不擴大實際威脅，卻消除一個不必要的資料永久損失風險。

同時 UnlockScreen 目前**沒有逃生口**：作用中身分一被鎖，忘記密碼就卡死在解鎖畫面，
只能清 app 資料重來（且重來會踩 ADR-0066 的命名空間陷阱、舊 `.enc` 淪為孤兒）。

## 考量的選項

- 選項 A：維持現況（忘記密碼＝本機資料永久損失）——安全但對使用者過苛，且與
  「已強制 nsec 備份」矛盾。
- 選項 B：**資料金鑰雙重包裹＋nsec 救援重設密碼**（採用）——見決策。
- 選項 C：把密碼雜湊存起來供「重設」——否決：等於引入一把繞過加密的旁路，
  與 ADR-0067「密碼即解密原料、非比對憑證」根本衝突。
- 選項 D：nsec 直接當第二把日常解鎖鑰匙（每次可用 nsec 解）——否決：養成日常
  貼 nsec 的壞習慣（同 ADR-0067 否決 nsec 日常登入的理由）；救援應是低頻例外。

## 決策

1. **資料金鑰雙重包裹**：啟用本地密碼時，`db:<namespace>` 金鑰除了被密碼 KEK 包裹
   （日常路徑），**另存一份被 nsec 衍生金鑰包裹**於 `rescue:<namespace>`。
   - nsec 衍生：`SHA-256(nsec ‖ "cinder-rescue-v1")`——nsec 已是 256-bit 高熵，
     用域分隔雜湊即可，**不需** Argon2 那種慢雜湊（那是給低熵密碼防暴力的）。
   - AEAD 重用 encstore（AES-256-GCM）。
2. **救援重設流程**（忘記密碼時）：使用者於解鎖畫面「忘記密碼？」輸入 nsec
   （或備份碼＋備份密碼，前端以 ADR-0070 解出 nsec）＋設**新**密碼 →
   nsec 解開 `rescue:` 拿回**真正那把** db 金鑰 → 以新密碼重包裹 nsec 與 db 金鑰
   （防斷電順序，同 pass_change）→ 解鎖進入。
3. **舊資料原封回來**：因救援解開的是**同一把 db 金鑰**，那台機器的完整本地歷史
   `.enc` 直接可讀——是「救回」，不是「重來」（差別於單純重新匯入 nsec，後者拿
   乾淨儲存＋網路補資料、舊 `.enc` 淪孤兒）。
4. **重設＝設新密碼、非找回舊密碼**：舊密碼永不可得（從未儲存）；救援後舊密碼作廢。
5. **向後相容（惰性補建）**：本功能前已啟用密碼的使用者無 `rescue:` blob；於下次
   成功 `pass_unlock`（此時 nsec 與 db 金鑰皆在手）時自動補建，無需手動重設。
6. **停用密碼時**一併清除 `rescue:` blob。

## 理由

- 救援要求**明文 nsec**。在有密碼的機器上，金鑰庫裡的 nsec 本身被密碼包裹——
  共用 OS 帳號的攻擊者拿不到 nsec，也就走不了救援；`rescue:` blob 對他只是解不開
  的密文。能用救援的只有握**獨立 nsec 備份**者＝既有的主金鑰持有者等級，非新開的洞。
- 因此本設計**不削弱** ADR-0067 對「共用機器」這個實際威脅的防護，只把「本機資料」
  的保護等級對齊到 nsec 本來就在的高度。
- nsec 高熵故救援衍生用快雜湊；日常仍走 Argon2id 密碼——分層正確。

## 後果

- 正面：忘記密碼不再等於本機資料永久損失（有 nsec/備份碼即可救回）；UnlockScreen
  有了逃生口；ADR-0067 的強制 nsec 備份從「只能重建身分」升級為「能救回全部本機資料」。
- 負面／已知殘餘風險：
  - 持有你**明文 nsec 備份**的人，現在也能解你這台機器的 `.enc`——但此人本就能
    讀你所有訊息與雲端快照，屬既有主金鑰持有者風險，非新增暴露面（文案須點明
    「nsec 備份的保管等級＝你全部資料的保管等級」）。
  - 連 nsec 備份都遺失＋忘記密碼 → 仍為本機資料永久損失（此時無任何主金鑰，
    合理且無解）。
- 修訂：ADR-0067「忘記密碼＝本機永久不可解」後果條款，改為「忘記密碼且**無 nsec
  備份**時永久不可解；持 nsec 備份可救援重設」。
- 後續行動 / 待辦：Rust `rescue_wrap/unwrap`＋IPC `pass_rescue`＋惰性補建、
  UnlockScreen 救援流程、i18n；完成後於 ARCHITECTURE.md 金鑰管理節註記。
