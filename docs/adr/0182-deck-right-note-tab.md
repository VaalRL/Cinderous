# 0182. 右欄「計算」分頁改為「便條」——計算為其功能之一

- 狀態：已接受（已實作）
- 日期：2026-07-17
- 相關文件：ADR-0079（三欄佈局·右側輔助區）、0097（算式計算機）、0134（對話背景·本機 UI 偏好）

## 背景與問題

右欄輔助區（`DeckRight`）原有一個「計算」分頁（ADR-0097）：單行輸入框，純本地算式、可插回
對話。使用者要求把它改為「**便條**」——一個更通用的私人便條區，**計算只是其中一個功能**
（便條裡輸入算式仍能自動辨識並算出）。

## 決策

### 1. 分頁改名：「計算」→「便條」（`AuxTab: "calc" → "note"`）

TABS 標籤改用 `aux_tabNote`（便條／Notes）。舊「計算」專屬字串（`aux_tabCalc`／
`calc_placeholder`／`calc_hint`／`calc_notExpr`）已無引用，一併移除；插入鈕字串
（`calc_insertExpr`／`calc_insertResult`）仍由便條的計算功能沿用。

### 2. `CalcPanel` → `NotePanel`：多行便條，計算為功能之一

- **每對話一張私人便條**：`note-store.ts`（`nb.note.<convoId>`，localStorage）——純本機、
  **不廣播、不上雲**，與 chatbg 等本機 UI 偏好同類。面板以 `key={activeId}` 重掛，切對話即載
  對應便條；空字串＝清鍵。
- **計算功能（ADR-0097 保留）**：取便條**最後一個非空行**判定是否為算式；是則即時顯示結果
  （禁用 eval，走 core `calcPreview`），可「插入表達式／插入結果」回主對話框（不動草稿）。
- 多行 `textarea`（可縱向調整）取代單行 input；CSS `.daux__note`/`.daux__notein`，計算輸出
  沿用 `.daux__calcout`。

## 後果

- 正面：右欄多了實用的「便條」——可記與該對話有關的私人筆記，跨開關持久；計算無縫內建為
  其一功能（輸入算式即算）。私隱：便條只存本機、不外流。
- 已知限制／取捨：
  - 便條存 localStorage **明文**（與 chatbg 等既有本機 UI 偏好一致；桌面 Tauri 另有 OS 層）。
    若日後要更嚴，可比照訊息走 encstore——目前依既有 UI 偏好慣例，不特別加密。
  - 只桌面右欄；行動端無此輔助區（不在本次範圍）。
  - 計算只認「最後一個非空行」——便條中段的算式不會即時算（避免多行歧義）；符合「便條為主、
    計算為輔」的定位。
- 測試：`note-store.test`（每對話隔離／空＝清鍵／SSR 安全）；`DeckRight.test` 更新為便條分頁
  入口（並確認舊 `aux-tab-calc` 已不存在）；i18n 8＋desktop 411 綠燈、typecheck 通過。
