# 0068. 管理員開機群組快照廣播（群組成員資格自癒）

- 狀態：已接受
- 日期：2026-07-09
- 相關文件：docs/adr/0009（多設備同步）、0027（群組加密）、0036（hint 陳舊與群訊帶 hint）、
  0041（節流外送匣）、0047/0049（組織名冊與群組）、0061（加密個人檔廣播）、
  0065（離線留言 TTL）、0066（搬家與開機廣播帶 hint）、0067（nsec 定位）、
  packages/core/src/group.ts、apps/desktop/src/backend/relay-backend.ts

## 背景與問題

nsec 匯入新裝置是認可的救援通道（ADR-0067），聯絡人靠「7 天離線窗＋開機個人檔
廣播」自我修復（ADR-0061/0066 H1），但**群組回不來**：

1. 群組是控制訊息驅動的狀態機（ADR-0027）：只有建群／加成員控制訊息會實例化
   群組；當初的控制訊息早已超過 7 天離線窗過期。
2. 防灌入設計讓陌生群訊被靜默丟棄（`receiveGroup` 查無此群即 return）——正確的
   安全性，但代價是新裝置期間的群訊**靜默漏信**，且群訊無送達回條，發送者無感知。
3. 一線生機已存在：收件匣訂閱每次啟動重放 7 天存貨，被丟棄的群訊未入庫、不會被
   去重擋掉——只要群組在窗內恢復並重啟一次，漏掉的群訊就補得回來。缺的只是
   「讓群組自動恢復」的權威訊息。

## 考量的選項

- 選項 A：維持現況（手動請管理員重邀＋重啟）——依賴人工，靜默損耗期不可控。
- 選項 B：**管理員開機群組快照廣播**（採用）——複製 ADR-0061 開機廣播的成功模式。
- 選項 C：任意成員可回應「群組狀態請求」——否決：成員代言群組＝偽造面，
  收端無法驗證非 admin 快照的權威性。
- 選項 D：relay 端保存群組狀態——否決：違反零伺服器狀態鐵則，且向中繼洩漏群組結構。

## 決策

1. **快照訊息**：群組控制訊息（kind 40）新增 `snapshot` 型別，內容＝群組 id、名稱、
   完整成員清單；Gift Wrap 逐成員加密、rumor 帶 relay hint（同 ADR-0036 群訊）、
   `expiration` now＋7 天（對齊 ADR-0065——離線成員的份存離線倉）。
2. **發送**：管理員 `start()` 在 `broadcastProfile()` 旁增 `broadcastGroups()`——
   對每個 `admin === 自己` 的群組、每位成員扇出，走節流外送匣（ADR-0041）；
   session 去重（同 `profileSentTo` 模式），另設「每群每日至多一次」節流。
3. **收端**：`applyControl` 增 snapshot 分支——
   - 本機已有該群：冪等對帳（名稱／成員以 admin 快照為準；非 admin 的快照沿用
     `applyGroupControl` 權威檢查拒絕）。
   - 查無此群：**僅當「Gift Wrap 驗證的發送者＝快照所稱 admin」**才實例化
     （seal 已密碼學驗證發送者，無法冒名）。
4. **排除企業組織群**（`org` 旗標）：簽章名冊（ADR-0047/0049）已是更強的權威對帳。

## 理由

- 全程搭現成便車：開機廣播模式（0061）、群控制管線（0027）、離線倉 TTL（0065）、
  外送匣（0041）、hint 順帶自癒（0036）——收端近乎零改動。
- 權威乾淨：admin 自己簽發自己群組的快照，成員無法代言、外人無法冒名。
- 與 7 天重放組成閉環：新裝置的群組靜默漏信收斂為「最多延遲至管理員下次開機的補收」。

## 後果

- 正面：nsec 換機後，活躍群組在管理員下次開機（或 7 天窗內存貨）自動回歸；
  群組路由 hint 每次開機刷新。
- 負面／已知殘餘風險：
  - 管理員本人換新裝置救不了自己的群（快照源頭是他）——正解仍是 ADR-0009 裝置同步。
  - 快照只含成員資格，不含群組歷史訊息。
  - 知道群組 id 的**前成員**可對白紙狀態的新裝置搶先塞假快照（id 只在成員密文間
    流通，外人拿不到）——與現行「任意人可發建群邀請」同級的既有攻擊面；緩解於
    實作時定案（恢復型群組 UI 標示、與真 admin 快照衝突時提示使用者）。
  - 成本＝自建群組數×成員數顆小包裹／開機，與 broadcastProfile 同量級，節流封頂。
- 後續行動 / 待辦：依 ROADMAP Phase H 的 H5 施工（core `snapshot` 控制型別＋後端
  `broadcastGroups()`＋收端分支，TDD）；完成後更新 ARCHITECTURE.md。
