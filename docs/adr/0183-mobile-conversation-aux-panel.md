# 0183. 行動端對話輔助面板——媒體／對話串／便條（桌面右欄的行動端版）

- 狀態：已接受（已實作）
- 日期：2026-07-17
- 相關文件：ADR-0079（桌面三欄右側輔助區）、0182（右欄便條）、0051（對話串）、0102（圖片縮圖）、
  0097（算式計算機）、0112（靜態加密）、0179（加密託管的先例）

## 背景與問題

桌面右側輔助區（`DeckRight`）有媒體相簿、對話串、便條等聚合檢視；行動端是單欄全螢幕、**無右欄
容器**，這些「聚合檢視」缺席（只有群組成員面板）。要補齊，硬搬三欄佈局不合行動端。

## 決策

### 1. 一顆 📋 鈕 → 分頁式輔助面板（不是三欄）

對話標頭加一顆 📋，展開一個輔助面板（比照現有成員面板），內含 **3 個分頁：媒體／對話串／便條**
——把桌面右欄的分頁概念，以行動端的展開面板呈現。分頁標籤帶計數（媒體/對話串）。

- **媒體相簿**（ADR-0102）：對話中可顯示的圖片縮圖網格；有原圖 blob 者點開新分頁看原圖。
- **對話串**（ADR-0051）：有回覆的串根＋回覆數清單（唯讀）。
- **便條**（ADR-0182 的行動端版）：見下。

### 2. 對話串邏輯上移共用引擎（消除重複）

`thread-util`（`replyCounts`/`mainMessages`/`threadMessages`/`rootIdOf`）原在桌面 `ui/`；行動端也
要用。依「不建立重複」上移到 `@cinder/engine`（`backend/thread-util.ts`），桌面兩處 importer
（`DeckRight`/`ConversationWindow`）改自 `@cinder/engine` 匯入，測試一併移入引擎。

### 3. 便條：每對話一張、**加密**落盤（比桌面嚴）

`apps/mobile/src/note-store.ts`：每對話一張私人便條（`nb.note.<convoId>`）。與桌面**明文**
localStorage 不同——行動端對本機資料一律更嚴（ADR-0112），故以**作用中身分 nsec 導出金鑰**
`sealValue` **加密落盤**（比照離職託管 ADR-0179）；App 層持金鑰、只把明文交給元件（`onNoteLoad`/
`onNoteSave`），密文離開該 nsec 就解不開。畫面以 `key={activeId}` 重掛 → 便條即載對應對話。

**計算不併入便條**：行動端計算已是**輸入框**的 inline 功能（ADR-0097，打字即浮算式 chip），
便條再放一個會重複，故行動端便條**只做筆記**——與桌面「計算為便條功能之一」刻意不同（各平台
取最順的形式，非不一致）。

## 後果

- 正面：行動端補齊桌面右欄的三項聚合檢視（媒體/對話串/便條），以單顆 📋＋分頁面板呈現，不動
  單欄佈局。便條加密落盤（比桌面更嚴）。thread-util 單一真實來源、桌面/行動共用。
- 已知限制／取捨：
  - 面板內容為互動展開（`auxOpen`）→ SSR 測試只驗入口鈕，內容渲染由邏輯/純函式測試把關
    （與成員/背景面板一致）。
  - 對話串為唯讀清單（點不跳轉）；媒體點開原圖走 `window.open`（web）。
  - 便條計算不內建（計算在輸入框）——刻意。
- 測試：`thread-util.test`（隨模組移入引擎，5 例）；行動端 `note-store.test`（加密往返/每對話隔離/
  密文不含明文/錯金鑰回空/SSR 安全）；`ConvoComposer.test` 補 📋 入口；engine 265＋mobile 197＋
  desktop 406＋i18n 8 全綠、四端 typecheck 通過。
