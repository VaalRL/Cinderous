# 0079. 桌面三欄整合佈局 ＋ 經典/新版一鍵切換

- 狀態：已接受
- 日期：2026-07-11
- 相關文件：docs/adr/0040（群組本地標籤——本 ADR 擴到聯絡人）、0051（對話串面板）、
  0074（前端分層，社群自訂前端）、apps/desktop/src/App.tsx、
  apps/desktop/src/ui/ContactListWindow.tsx、ConversationWindow.tsx、msn.css

## 背景與問題

桌面版現為 **MSN 浮動視窗**模型（`.desktop` flex-wrap 平鋪多個獨立視窗：聯絡人視窗 ＋
`open[]` 展開的 N 個對話視窗）。使用者要新增一種 **整合式三欄佈局**（左：聯絡人＋群組列；
中：對話區；右：對話輔助區），且要**與舊佈局一鍵切換**（非先前否決的 N-候選切換器，而是
**二選一**：經典 ↔ 新三欄）。

## 決策

1. **兩種佈局共存、`nb.layout` 一鍵切換**（`classic` | `modern`，本地儲存）；**預設 `classic`**
   （不驚嚇現有使用者，三欄成熟後再議是否換預設）。設定面板加切換。
2. **同一套狀態、同一批內容元件，只換外殼**（降風險核心）：
   - 狀態：`open[]`（開啟中的對話）＋新增 `active`（當前）。經典忽略 `active`（全展成浮動窗）；
     三欄以 `open[]` 為分頁列、`active` 決定中間顯示。加 `active` 不破壞經典。
   - 內容：`ConversationWindow`、聯絡人列、thread/成員/相簿邏輯**全部重用**；差別只在外框——
     經典＝帶標題列浮動窗，三欄＝無邊框內嵌（`ConversationWindow` 加 embedded 模式）＋搬進側欄。
3. **三欄結構**（CSS Grid，左右可收合、各自捲動；頂部 idbar 跨欄）：
   - **左側欄**：聯絡人＋群組**混合一列、預設依最近互動時間排序**（用各對話最後訊息時間；
     無互動者殿後）。上半**搜尋（名稱 ＋ 訊息內容）＋自訂標籤篩選**。雙擊 → 中間開分頁。
   - **中間**：對話**分頁列** ＋ 單一內嵌 active 對話。
   - **右側欄**：分頁 **對話串 Threads／成員 Members／媒體 Media／資訊 Info**（初始四頁，後續擴充）。
4. **自訂標籤擴到聯絡人**（擴充 ADR-0040）：聯絡人比照群組可貼本地標籤（本地儲存），
   讓左側欄標籤篩選同時作用於聯絡人與群組。
5. 分階段（**經典全程不動、加法**，每階段測試綠可獨立提交）：
   - Q1 `nb.layout` 開關 ＋ 殼選擇器 ＋ 三欄骨架（先接現有元件）。
   - Q2 左側欄：最近互動排序、搜尋（名稱＋內容）、標籤篩選（含擴標籤到聯絡人）、雙擊開分頁。
   - Q3 中間：`open[]` 分頁列 ＋ `ConversationWindow` 內嵌模式。
   - Q4 右側欄：Threads／Members／Media／Info 分頁殼，搬入現有面板。

## 理由

- 二佈局共存是「2 結構 × 共用內容」的精簡實現：三欄為**加法**，經典原封不動 → 回歸風險低於
  「整個取代」。呼應 ADR-0074（同一前端內建多佈局，是其輕量版）。
- 純前端/版面層變更，不動訊息、加密、後端；254 測試涵蓋的功能邏輯不受影響。

## 後果

- 正面：使用者可選經典或現代三欄；三欄達成搜尋/標籤/最近互動/分頁/右側輔助的現代體驗。
- 負面／已知殘餘：長期維護**兩套外殼**（每個新 UI 功能要顧兩種佈局）；三欄「一次一個 active
  對話」失去經典的多窗並排。標籤/佈局偏好**本地儲存、不隨換機同步**（比照 ADR-0040/0064）。
- 後續：右側欄分頁之後續擴充；三欄成熟後再評估是否換預設。
