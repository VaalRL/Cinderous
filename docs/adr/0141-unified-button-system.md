# 0141. 統一按鈕系統：主色實心 / 中性描邊 / 危險紅，形狀與焦點一致

- 狀態：已接受（已實作）
- 日期：2026-07-15
- 相關文件：ADR-0078（主色 --accent 主題化）、0080（設計 token）、0139（自訂對話框，按鈕語意來源之一），
  apps/desktop/src/ui/msn.css

## 背景與問題

桌面按鈕風格長期分歧：

- **主要動作有兩種主色**：登入／送出／建立／AI 改寫／建群／導出用**綠色漸層**
  （`#8ad08a→#2f9e44`／`#3fae5a`），但對話框確認鈕用 **`--accent` 實心**——同一種「主要動作」卻兩種顏色。
- **中性小按鈕約十種變體**：圓角 4/5/6/10px、內距 1–4px、字級 10–13px、文字色 ink/accent/muted 各不同。
- **危險色兩種紅**：`#e5484d` 與 `#b3261e`。
- **未指定 class 的 `<button>`**（配對／中繼站／名冊面板）落回**瀏覽器灰色預設**。
- 多數按鈕**沒有一致的鍵盤焦點環**。

## 決策

一套三語意的統一按鈕系統（純 CSS，不動 testID／行為）：

1. **基底**：為 `<button>` 元素設中性描邊預設（`--field` 底、`--border` 框、6px 圓角、一致內距與字重）
   ——**任何未指定 class 的按鈕不再落回 OS 灰**；有 class 者 specificity 較高、各自覆寫。
2. **主要動作＝主色實心**（使用者選定）：所有主要動作統一 `background: var(--accent); color:#fff`，
   **隨主題色變**、與標題列與對話框一致。移除所有綠色漸層與 `#3fae5a`。涵蓋
   `.signin button`／`.composer__send`／`.pill`／`.addbar__row button`／`.dialog__btn`／
   `.rewrite__go`／`.rewrite__adopt`／`.groupmodal__create`／`.export__go`／`.lightbox__btn`。
3. **中性／次要＝描邊淺底**：`--field` 底、`--border` 框、hover 進 `--hover`／主色框；圓角一律 6px
   （4/5px outlier 收斂）。
4. **危險＝單一紅** `#e5484d`（統一 `.dialog__btn--danger` 與 `.settings__staleact--danger`）。
5. **全域**：所有按鈕一致的 `:focus-visible` 主色焦點環與 `:disabled` 態。
6. **語意色保留**：`.callbtn--accept` 的綠（接聽）與 decline 的紅屬**語意色**（接/掛電話的通用語彙），
   **不**併入通用主色系。

## 理由

- 「主要動作」該有**單一顏色語言**；主色實心既與最新元件（對話框）一致，又吃 `--accent` 隨主題色變
   ——比孤立的綠色漸層更一致、更可主題化（使用者在設定選森綠/葡萄紫…按鈕也跟著變）。
- 基底 `<button>` 樣式是「統一」的最後一哩：以往靠各自 class，漏網的裸按鈕就露出 OS 灰；給元素級預設，
   保證零漏網、且以 specificity 讓既有 class 全數覆寫、不回歸。
- 語意色（接聽綠/掛斷紅）與品牌主色是**不同軸**，不應被「統一」抹平。

## 後果

- 正面：
  - 主要動作**單一主色、隨主題變**；中性、危險、焦點、停用態全一致；裸按鈕不再灰。
  - 純 CSS，零 TSX/testID/行為改動；既有測試不受影響（測 testID 與行為，不測樣式）。
- 已知限制：
  - 部分**裸按鈕**（如面板的「套用/取消」）現在都吃中性描邊預設——視覺一致，但該情境的「主要 vs 次要」
    層級未逐顆標示；若要強調某顆為主要動作，之後個別加 primary class（非本次目標）。
  - 綠→主色是明顯的視覺轉變；若日後想保留部分懷舊綠，屬另一個主題化決策。
