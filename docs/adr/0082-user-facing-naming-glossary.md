# 0082. 使用者可見名稱統一詞彙表（登入／同步／搬家／備份／配對）

- 狀態：已接受
- 日期：2026-07-12
- 相關文件：ADR-0066（更換中繼站）、0069（relay 自動搬遷）、0070（加密備份碼）、0071（加密雲端快照）、0072（配對克隆 D4a）、0073（忘記密碼救援）、0081（行動端登入）；`packages/i18n/src/messages.ts`

## 背景與問題

專案演進中累積了多條「登入／資料同步／搬家／換機／備份還原／配對」流程，各自在不同時期命名，導致**同一個詞多義、同一件事多名**，使用者可見文案尤其混亂：

- 「**搬家**」同時指：更換中繼站（0066）、relay 自動搬遷（0069）、換裝置（手冊）——三件不同的事。
- 「**快照 / 同步 / 備份**」三詞混用於同一個雲端功能（標籤「雲端同步」＋副標「加密快照」＋按鈕「立即備份」）。
- 同一個**配對克隆**流程，新機入口在桌面叫「從舊裝置匯入」、在行動端叫「配對匯入」。
- **nsec** 有「金鑰／私鑰／key／secret key」四種叫法。
- 「把資料弄回來」動詞混用：還原／救援／匯入。

## 考量的選項

- **選項 A（採用）：只統一「使用者可見名稱」**——i18n 文案值＋少數硬編碼中文；不改程式識別字、不改 ADR/文件既有措辭。低風險、不動邏輯與測試契約。
- 選項 B：連程式識別字（函式/型別/i18n 鍵前綴）一併對齊——全面但改動面大、需重跑大量測試。
- 選項 C：連 ADR/ROADMAP/手冊/PRD 措辭全部對齊——最徹底、工程量最大。

## 決策

**採選項 A：統一使用者可見名稱，並以本 ADR 的「詞彙表」為 SSOT。** 未來新增 UI 文案一律遵循。

### 統一詞彙表（依「你在搬什麼」分類）

| 概念 | 統一名稱（繁中 / 英文） | 取代掉的舊說法 |
| --- | --- | --- |
| nsec 私鑰 | **私鑰（nsec）** / secret key (nsec) | 金鑰、key |
| 換裝置·P2P·舊機端 | **配對新裝置** / Pair a new device | （不變） |
| 換裝置·P2P·新機端 | **從舊裝置匯入** / Import from old device（桌面＋行動一致） | 「配對匯入」（行動）|
| 換裝置（泛指把身分搬到另一台） | **換裝置** / switch device | 換機 |
| 更換中繼站·手動 | **更換中繼站** / Change relay | 搬家 |
| 更換中繼站·自動（退役/離線） | **自動更換中繼站** / auto-change relay | 硬編碼「自動搬家」|
| 保護資料·顯示 nsec | **身分備份** / Identity backup | （不變）|
| 保護資料·加密字串 | **加密備份碼** / encrypted backup code | （不變）|
| 保護資料·存中繼 | **雲端備份** / Cloud backup | 雲端同步、加密快照 |
| 救回資料·一般 | 動詞 **還原** / restore | 匯入 / apply |
| 救回資料·忘記密碼 | **救援** / recover（專用於此情境） | （界定清楚）|

原則：**搬去別台裝置（換裝置）／換另一座中繼站（更換中繼站）／救回自己的資料（還原·救援）三者用詞完全分開，不再共用「搬家」。** 「同步」一詞自此**保留給未來的即時 delta（D4b）**，不再指雲端備份。

### 本次實作範圍（已套用）

- i18n：`settings_cloud*`（雲端同步/快照 → 雲端備份）、`msignin_*`/`mpair_*`（金鑰→私鑰、配對匯入→從舊裝置匯入、舊機→舊裝置）、`settings_backupCodeHint`（換機→換裝置）。
- 硬編碼：`App.tsx` relay 自動搬遷通知「自動搬家」→「自動更換」。
- 對應測試斷言同步更新。

### 明確不在本次範圍（保留現況）

- **程式識別字**（`signIn_`/`msignin_` 鍵前綴、`runPairingSource` vs `runPairSource`、`cloudSync`、`snapshot` 等）維持不變——屬選項 B。
- **ADR/ROADMAP/手冊/PRD** 既有措辭不追改——屬選項 C。
- 「**排水**（drain）」為獨有內部術語、非命名衝突，UI 維持「排水」；白話化（如「舊站續收期」）另議。

## 後果

- 正面：使用者面對的登入/備份/換裝置/換中繼站用詞一致、彼此不再混淆；「同步」語意釋出給未來 delta。
- 負面 / 已知殘餘：UI 文案與程式識別字暫時「詞面不一致」（如按鈕「雲端備份」對應 `cloudSync`/`snapshot` 識別字）——可接受，識別字統一列為 B 待辦。
- 後續：如採選項 B/C，以本詞彙表為準批次對齊識別字與文件；行動端 app 殼與後續新畫面文案一律遵循本表。
